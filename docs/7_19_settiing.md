# 설정 시스템 기획서

## 1. 시스템 개요

### 1-1. 목적

설정 시스템은 유저가 **게임 플레이 환경을 자신의 기기와 취향에 맞게 조정**할 수 있도록 제공하는 편의 기능이다.
본 시스템은 **전투 밸런스나 성장 결과에는 영향을 주지 않으며**, 사운드/그래픽/알림/계정/지원 정보 관리에 초점을 둔다.

### 1-2. 설계 방향

* 방치형 게임 특성상 **메뉴 구조를 최대한 단순화**한다.
* 설정 변경은 가능한 한 **즉시 반영**한다.
* 전투 중에도 설정 창 진입이 가능하되, **전투 진행 자체는 멈추지 않는다**.
* 계정 단위로 유지해야 하는 값과, 기기 단위로 유지해야 하는 값을 분리한다.
* 설정 항목은 **필수 편의 기능만 제공**하며, 복잡한 세부 옵션은 제외한다.

### 1-3. 제공 범위

설정 시스템은 아래 기능만 포함한다.

1. 사운드 설정

   * 전체 음량
   * BGM 음량
   * 효과음 음량
   * 진동 ON/OFF

2. 화면/성능 설정

   * FPS 설정(30 / 60)
   * 절전 모드 ON/OFF
   * 화면 흔들림 ON/OFF
   * 데미지 텍스트 표시 ON/OFF

3. 알림 설정

   * 푸시 알림 전체 ON/OFF
   * 오프라인 보상 최대 도달 알림
   * 던전 티켓 최대 충전 알림
   * 일일 초기화 알림
   * 시즌패스/이벤트 보상 수령 가능 알림

4. 기타 설정

   * 언어 설정(한국어 / 영어)
   * 계정 연동 상태 확인
   * UID 확인 및 복사
   * 버전 정보 확인
   * 이용약관 / 개인정보처리방침 / 고객센터 진입
   * 기본값으로 초기화

### 1-4. 비포함 범위

아래 기능은 설정 시스템 범위에서 제외한다.

* 쿠폰 입력
* 커뮤니티 링크
* 캐릭터 삭제
* 마케팅 수신 동의
* 상세 접근성 기능(글자 크기, 색약 모드 등)
  → 추후 필요 시 별도 확장

## 2. 규칙 설명

### 2-1. 진입 규칙

* 메인 화면 우측 상단의 **설정 버튼(톱니바퀴)** 을 통해 진입한다.
* 전투 화면에서도 진입 가능하다.
* 설정 창은 **팝업/오버레이 방식**으로 열리며, 닫으면 이전 화면으로 복귀한다.

### 2-2. 적용 규칙

* 대부분의 설정은 변경 즉시 반영한다.
* 단, **언어 변경**은 현재 열려 있는 UI를 새로고침한 뒤 반영한다.
* **사운드/그래픽/진동** 관련 옵션은 기기 즉시 적용 후 로컬 저장한다.
* **알림/언어/계정 상태** 관련 옵션은 서버 동기화 대상이다.

### 2-3. 기본값

초기 설치 시 기본값은 아래와 같다.

* 전체 음량: 100
* BGM 음량: 70
* 효과음 음량: 80
* 진동: ON
* FPS: 60
* 절전 모드: OFF
* 화면 흔들림: ON
* 데미지 텍스트: ON
* 언어: 기기 언어 자동 감지

  * 지원 언어가 아니면 한국어
* 푸시 알림: ON
* 오프라인 보상 알림: ON
* 던전 티켓 알림: ON
* 일일 초기화 알림: ON
* 시즌패스/이벤트 알림: ON

### 2-4. 변경 규칙

* 슬라이더 값 변경 시 즉시 반영한다.
* 토글 버튼 변경 시 즉시 저장한다.
* FPS 변경은 즉시 적용하되, 기기 성능 이슈가 있을 경우 시스템이 강제로 30FPS로 낮출 수 있다.
* 절전 모드가 ON이면 다음 정책을 우선 적용한다.

  * FPS 30 고정
  * 일부 이펙트 최소화
* 푸시 알림 전체 OFF 시 하위 알림 항목은 모두 비활성화된다.
* 기본값 초기화 시 현재 설정값을 모두 기본값으로 되돌린다.

### 2-5. 계정 관련 규칙

* 설정 화면에서 현재 계정 상태를 표시한다.

  * 게스트
  * Google 연동
  * Apple 연동
* 게스트 계정은 **계정 연동 유도 문구**를 노출한다.
* 로그아웃은 연동 계정에서만 허용한다.
* 게스트 상태에서는 로그아웃 대신 **계정 연동 버튼**만 제공한다.

### 2-6. 알림 관련 규칙

* 푸시 알림은 **OS 권한 허용**이 선행되어야 한다.
* 게임 내에서 푸시 ON으로 설정했더라도 OS 권한이 꺼져 있으면 실제 발송되지 않는다.
* OS 권한이 차단된 상태에서 푸시 ON 시도 시, **기기 설정 이동 안내 팝업**을 띄운다.

## 3. 데이터 구조

설정 데이터는 **기기 로컬 저장 데이터**와 **서버 동기화 데이터**로 나눈다.

### 3-1. 로컬 저장 데이터

기기 성능과 표현 방식은 기기마다 달라질 수 있으므로 로컬에 저장한다.

```json
LocalSettings {
  master_volume: 100,
  bgm_volume: 70,
  sfx_volume: 80,
  vibration_on: true,
  fps_mode: 60,
  battery_saver_on: false,
  screen_shake_on: true,
  damage_text_on: true,
  local_updated_at: "2026-04-03T22:00:00Z"
}
```

### 3-2. 서버 저장 데이터

계정 단위로 유지되어야 하는 값은 서버에 저장한다.

```json
UserSettings {
  user_id: 123456,
  language: "ko",
  updated_at: "2026-04-03T22:00:00Z"
}
```

### 3-3. 기기별 알림 설정 데이터

알림은 계정보다 **기기 단위**로 관리하는 것이 적절하다.
같은 계정을 여러 기기에서 사용할 수 있기 때문이다.

```json
UserDeviceSettings {
  user_id: 123456,
  device_id: "A1B2C3D4",
  push_enabled: true,
  push_offline_reward: true,
  push_dungeon_ticket_full: true,
  push_daily_reset: true,
  push_pass_event_reward: true,
  push_permission_state: "granted",
  push_token: "xxxx",
  updated_at: "2026-04-03T22:00:00Z"
}
```

### 3-4. 추천 enum 정의

```text
language: ko / en
fps_mode: 30 / 60
push_permission_state: granted / denied / not_determined
account_type: guest / google / apple
```

### 3-5. 저장 우선순위

* 로컬 설정: **클라이언트 기준**
* 언어 설정: **서버 기준**
* 알림 설정: **기기별 서버 데이터 기준**

## 4. UI 구성

### 4-1. 기본 구조

설정 화면은 **단일 스크롤 팝업** 구조로 구성한다.
서브 메뉴를 만들지 않고 한 화면에서 모든 설정을 관리한다.

### 4-2. UI 섹션 구성

#### A. 상단 영역

* 제목: 설정
* 닫기 버튼
* UID 표시
* UID 복사 버튼
* 현재 버전 표시

#### B. 계정 영역

* 현재 계정 상태 표시

  * 게스트 / Google / Apple
* 계정 연동 버튼
* 로그아웃 버튼(연동 계정만 노출)

#### C. 사운드 영역

* 전체 음량 슬라이더
* BGM 음량 슬라이더
* 효과음 음량 슬라이더
* 진동 토글

#### D. 화면/성능 영역

* FPS 선택 버튼(30 / 60)
* 절전 모드 토글
* 화면 흔들림 토글
* 데미지 텍스트 토글

#### E. 알림 영역

* 푸시 알림 전체 토글
* 오프라인 보상 최대 알림 토글
* 던전 티켓 최대 알림 토글
* 일일 초기화 알림 토글
* 시즌패스/이벤트 보상 알림 토글

#### F. 기타 영역

* 언어 선택 드롭다운
* 기본값으로 초기화 버튼
* 이용약관 버튼
* 개인정보처리방침 버튼
* 고객센터 버튼

### 4-3. UI 정책

* 토글형 옵션은 우측 정렬
* 슬라이더는 값이 바로 보이도록 수치 표기
* 비활성화된 알림 항목은 회색 처리
* 외부 링크 버튼은 새 WebView 또는 외부 브라우저로 연결
* 위험한 행동(로그아웃, 초기화)은 확인 팝업 노출

### 4-4. 확인 팝업

다음 기능은 확인 팝업을 사용한다.

* 기본값 초기화
* 로그아웃
* OS 설정 화면 이동 안내
* 외부 링크 진입

## 5. 예외 처리

### 5-1. 네트워크 끊김 상태

* 로컬 설정은 정상 반영 및 저장한다.
* 서버 저장 대상 설정(언어, 알림)은 **변경 내용을 임시 큐에 보관**한다.
* 재접속 시 서버에 재전송한다.

### 5-2. OS 알림 권한 거부 상태

* 게임 내 푸시 토글을 ON해도 실제 발송은 불가능하다.
* “기기 설정에서 알림 권한을 허용해 주세요” 팝업 노출
* 하위 알림 토글은 보이되, 상태 설명 문구를 함께 표시한다.

### 5-3. 설정 데이터 손상

* 로컬 설정 파일 파싱 실패 시 기본값으로 복구한다.
* 복구 시 1회 로그를 남긴다.
* 복구 직후 “설정값이 초기화되었습니다” 토스트 노출 가능

### 5-4. 다중 기기 접속

* 언어는 최근 수정 시간을 기준으로 최신값 반영
* 알림은 device_id 기준으로 분리 관리
* 사운드/그래픽 옵션은 기기별 로컬 유지

### 5-5. 언어 리소스 누락

* 선택 언어의 일부 문구가 없을 경우 한국어를 fallback으로 사용한다.
* 문자열 키 누락은 에러로 기록한다.

### 5-6. 계정 연동 실패

* 네트워크 오류 또는 플랫폼 인증 실패 시 기존 상태 유지
* 실패 팝업 노출
* 중복 연동/잘못된 토큰은 서버 로그 기록

### 5-7. 구버전 클라이언트

* 특정 설정 항목이 구버전에서 지원되지 않을 경우 서버는 해당 필드를 무시한다.
* 클라이언트는 자신이 지원하는 필드만 전송/표시한다.

## 6. 서버/클라이언트 처리 구분

## 6-1. 클라이언트 처리

클라이언트가 담당하는 항목

* 설정 화면 표시 및 입력 처리
* 슬라이더/토글/드롭다운 UI 반응
* 사운드 즉시 적용
* 진동 ON/OFF 적용
* FPS/절전 모드 적용
* 화면 흔들림/데미지 텍스트 표시 여부 적용
* 로컬 저장 및 로컬 로드
* 서버 동기화 요청 전송
* OS 알림 권한 상태 확인

## 6-2. 서버 처리

서버가 담당하는 항목

* 언어 설정 저장
* 기기별 알림 설정 저장
* 푸시 토큰 저장 및 활성 상태 관리
* 계정 연동 상태 반환
* UID 및 계정 메타 정보 제공
* 설정 변경 이력 로그 기록
* 푸시 발송 대상 필터링

## 6-3. 공통 처리

* 설정 변경 시점의 updated_at 관리
* 다중 기기 충돌 처리
* 버전 호환성 체크

## 6-4. 권장 기준

* **로컬 우선**: 사운드, 성능, 표현
* **서버 우선**: 언어, 계정 상태
* **기기별 서버 저장**: 푸시 알림

## 7. 로그 및 운영 포인트

### 7-1. 주요 로그 이벤트

아래 이벤트를 로그로 기록한다.

```text
settings_open
settings_close
settings_change
settings_reset_default
language_change
push_toggle_change
push_permission_result
account_link_try
account_link_success
account_link_fail
logout_try
logout_success
logout_cancel
support_open
terms_open
privacy_open
uid_copy
```

### 7-2. settings_change 상세 파라미터 예시

```json
{
  "event_name": "settings_change",
  "user_id": 123456,
  "device_id": "A1B2C3D4",
  "setting_key": "fps_mode",
  "before_value": 60,
  "after_value": 30,
  "location": "settings_popup"
}
```

### 7-3. 운영 확인 포인트

운영에서는 아래 수치를 지속적으로 본다.

* 전체 유저 대비 푸시 알림 허용률
* 화면 흔들림 OFF 비율
* 60FPS 사용 비율
* 절전 모드 ON 비율
* 계정 연동 완료율
* 설정 초기화 사용 빈도
* 언어 변경 비율
* 알림 권한 거부율
* 계정 연동 실패율
* 구버전 클라이언트 비중

### 7-4. 운영 활용 방법

* 화면 흔들림 OFF 비율이 높으면 기본값을 OFF로 조정 검토
* 절전 모드 사용률이 높으면 기본 성능 정책 완화 검토
* 계정 연동률이 낮으면 계정 영역 UX 개선
* 알림 허용률이 낮으면 알림 요청 시점 및 문구 개선
* 특정 패치 이후 설정 초기화 로그가 급증하면 데이터 손상 여부 점검

### 7-5. CS 대응용 정보

고객 문의 대응을 위해 아래 정보는 쉽게 확인 가능해야 한다.

* UID
* 계정 연동 상태
* 클라이언트 버전
* 마지막 설정 동기화 시각
* 푸시 토큰 등록 여부

# 정리

이 설정 시스템은 **전투 밸런스와 무관한 편의 기능만 제공하는 단순한 구조**로 설계한다.
핵심은 **즉시 반영, 단일 화면, 기기/계정 데이터 분리, 운영 추적 가능성**이다.

원하면 다음으로 내가 이어서 **같은 형식으로 “재화 확인” 또는 “오프라인 보상” 기획서**도 바로 맞춰서 써줄게.
