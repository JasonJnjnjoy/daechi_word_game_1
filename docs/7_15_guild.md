# #29 길드 시스템 기획서

## 1. 시스템 개요

### 1-1. 시스템 목적

길드는 유저에게 **소속감**, **일일 접속 동기**, **추가 보상 회수 루프**를 제공하는 사회형 시스템이다.
본 게임의 길드는 직접 전투 콘텐츠가 아니라, 다음 기능에 집중한다.

* 길드 가입/생성
* 길드 출석
* 길드 기부
* 길드 레벨 및 길드 버프
* 길드 상점
* 길드 랭킹

### 1-2. 코어 루프 내 역할

길드는 메인 루프를 방해하지 않고 보조하는 구조로 설계한다.

* 지역 진행으로 전투력 성장
* 성장한 전투력으로 길드 랭킹 기여
* 길드 활동으로 길드 코인 획득
* 길드 상점에서 추가 성장 재화 확보
* 길드 버프로 골드 수급/전투 효율 보조

즉, 길드는 **메인 성장 루프의 외곽에서 추가 보상을 제공하는 허브 시스템**이다.

### 1-3. 오픈 조건

* `지역 5 클리어` 또는 별도 지정한 메인 진행 조건 달성 시 오픈
* 오픈 전에는 길드 아이콘 비노출
* 오픈 시 튜토리얼 팝업 1회 노출

### 1-4. 1차 범위 포함 / 제외

포함:

* 길드 생성/가입/탈퇴/추방
* 길드 출석/기부
* 길드 레벨/버프
* 길드 상점
* 길드 랭킹

제외:

* 길드전
* 길드 보스
* 길드 연구/테크트리
* 실시간 길드 채팅
* 길드 전용 던전
* 길드 우편 분배 시스템

실시간 채팅 대신 **공지 1줄 + 최근 활동 로그**만 제공한다.

## 2. 규칙 설명

## 2-1. 길드 생성 / 가입 / 탈퇴

### 생성

* 유저는 다이아를 소모해 길드를 생성할 수 있다.
* 예시 비용: `다이아 500`
* 입력 항목:

  * 길드명
  * 소개글
  * 가입 방식(자동 가입 / 승인 가입)
  * 최소 전투력 조건
  * 길드 엠블럼

### 가입

* 유저는 한 번에 하나의 길드에만 소속될 수 있다.
* 길드 목록에서 검색 또는 추천 길드를 통해 가입 가능
* 가입 방식:

  * **자동 가입**: 조건 충족 시 즉시 가입
  * **승인 가입**: 신청 후 길드장/부길드장 승인 필요

### 탈퇴

* 탈퇴 즉시 길드 버프 제거
* 탈퇴 후 `24시간 재가입 대기시간` 적용
* 탈퇴 시 개인이 보유한 **길드 코인**은 유지
* 단, 주간 랭킹 보상 수령 자격은 초기화

### 추방

* 길드장/부길드장은 일반 멤버를 추방할 수 있다.
* 추방된 유저도 동일하게 `24시간 재가입 대기시간` 적용

## 2-2. 길드 역할 구조

역할은 3단계만 둔다.

### 길드장

* 공지 수정
* 가입 승인/거절
* 가입 조건 변경
* 멤버 추방
* 부길드장 임명/해제
* 길드장 위임
* 길드 해산

### 부길드장

* 가입 승인/거절
* 일반 멤버 추방
* 공지 수정

### 일반 멤버

* 출석
* 기부
* 길드 상점 이용
* 랭킹 보상 수령

### 인원 수

* 기본 최대 인원: `20명`
* 길드 레벨 상승에 따라 증가
* 최대 상한: `30명`

---

## 2-3. 길드 출석

### 기본 규칙

* 1일 1회 출석 가능
* 서버 기준 일일 리셋 후 초기화
* 출석 시:

  * 개인 보상 지급
  * 길드 경험치 증가
  * 개인 주간 기여도 증가

### 보상 방향

출석 보상은 단순하게 유지한다.

* 개인 보상: `길드 코인`
* 길드 보상: `길드 EXP`
* 개인 기록: `주간 기여도`

### 설계 의도

길드에 들어온 유저가 매일 최소 1회는 길드 화면을 방문하도록 유도한다.

---

## 2-4. 길드 기부

### 기본 규칙

기부는 2종만 운영한다.

1. 골드 기부
2. 다이아 기부

### 골드 기부

* 1일 1회 가능
* 소모 골드는 진행 구간에 맞게 스케일링하거나 고정값 사용
* 지급:

  * 길드 코인
  * 길드 EXP
  * 개인 주간 기여도

### 다이아 기부

* 1일 1회 가능
* 골드 기부보다 보상이 큼
* 지급:

  * 길드 코인
  * 길드 EXP
  * 개인 주간 기여도

### 설계 포인트

* 과금 유저만 유리하지 않도록 골드 기부만으로도 의미 있는 참여가 가능해야 함
* 다이아 기부는 효율형 선택지로 설계
* 기부로 별도 신규 재화를 만들지 않음

---

## 2-5. 길드 레벨 / 길드 버프

### 길드 레벨

길드 EXP가 누적되면 길드 레벨이 상승한다.

### 길드 EXP 획득처

* 길드 출석
* 길드 기부

### 길드 레벨 효과

길드 레벨 상승 시 다음 중 일부 또는 전부를 제공한다.

* 최대 인원 증가
* 길드 버프 수치 상승
* 길드 상점 상품 해금

### 길드 버프

길드 버프는 별도 강화 UI 없이 **길드 레벨에 따라 자동 적용**된다.

권장 버프 항목:

* 공격력 %
* HP %
* 골드 획득량 %

이 3개만 사용하면 단순하면서도 성장 체감이 있다.

### 적용 범위

* 길드 가입 중일 때만 적용
* 탈퇴/추방 즉시 해제
* 오프라인 보상에도 적용 가능
* 단, **가입 이전에 누적된 오프라인 시간에는 소급 적용하지 않음**

---

## 2-6. 길드 상점

### 개요

길드 상점은 길드 코인을 소비해 성장 재화를 교환하는 곳이다.

### 사용 재화

* `길드 코인`

### 판매 품목 예시

* 골드 묶음
* 장비 소환권
* 스킬 소환권
* 던전 티켓

### 운영 규칙

* 상품마다 `일일 제한` 또는 `주간 제한` 적용
* 일부 상품은 길드 레벨 조건으로 해금
* 랜덤 갱신, 수동 새로고침 기능 없음
* 판매 품목 수는 적게 유지하여 판단 피로를 줄임

### 설계 의도

길드 코인이 **명확한 소비처**를 가져야 길드 참여 동기가 유지된다.
상점은 복잡한 성장 분기 대신, 메인 성장 루프를 돕는 보조 공급처 역할에 집중한다.

---

## 2-7. 길드 랭킹

### 랭킹 기준

길드 랭킹은 **총 길드 전투력** 기준으로 운영한다.

* 각 멤버의 전투력을 합산
* 서버에서 캐시된 길드 총 전투력 기준으로 정렬
* 실시간 완전 동기화보다, 주기적 갱신 방식 사용

### 정산

* 주간 단위 정산
* 정산 시점의 길드 순위 기준으로 보상 확정

### 보상

길드 랭킹 보상은 개인 지급 방식으로 처리한다.

예시 보상:

* 다이아
* 장비 소환권
* 스킬 소환권
* 던전 티켓

### 보상 수령 자격

길드 갈아타기 악용 방지를 위해 아래 조건을 둔다.

* 정산 시점 기준 길드 소속 상태
* 가입 후 최소 24시간 경과
* 해당 주간 기여도 1 이상

### 동점 처리

동점 시 우선순위는 다음과 같다.

1. 먼저 해당 전투력에 도달한 길드
2. 길드 생성 시각이 더 빠른 길드

---

## 2-8. 길드 해산 / 위임

### 길드장 탈퇴

* 길드장이 탈퇴하려면 먼저 다른 멤버에게 길드장을 위임해야 함
* 혼자 남아 있는 경우에만 즉시 해산 가능

### 길드 해산

* 길드장만 가능
* 해산 시:

  * 모든 멤버 길드 소속 해제
  * 길드 버프 제거
  * 미수령 랭킹 보상은 정책에 따라 지급 또는 소멸 처리
* 해산 전 확인 팝업 2중 노출 권장

### 장기 미접속 길드장

* 길드장이 일정 기간 이상 미접속 시 자동 위임 가능
* 예: 7일 이상 미접속 시 부길드장에게 위임 후보 활성화

## 5. 예외 처리

### 중복 출석 요청

* 버튼 연타 또는 네트워크 재전송 시 서버에서 1회만 인정
* 이미 수령했다면 보상 미지급 + 에러 코드 반환

### 중복 기부 요청

* 서버에서 일일 기부 여부 검증
* 중복 요청은 차단

### 재화 부족

* 골드/다이아/길드 코인 부족 시 요청 거절
* 부족 재화 안내 팝업 노출

### 길드 정원 초과

* 신청 승인 순간 정원이 찼다면 가입 실패 처리
* 자동 가입도 동일하게 서버 최종 검증

### 다중 길드 가입 충돌

* 다른 길드 승인과 동시에 처리되는 경우 서버에서 먼저 확정된 1건만 인정

### 길드장 탈퇴 시도

* 위임하지 않은 상태에서는 탈퇴 불가
* 단독 길드일 경우 해산 유도

### 길드 해산 직후 화면 체류

* 클라이언트는 길드 해산 수신 즉시 미가입 화면으로 전환

### 랭킹 보상 악용

* 가입 직후 수령 방지를 위해 최소 가입 시간 + 최소 기여도 조건 적용

### 상점 리셋 경계 구매

* 구매 요청 시 서버 기준 시각으로 판단
* 리셋 직전/직후 경계에서도 서버 응답이 최종 기준

### 욕설/금칙어 길드명

* 생성 불가 처리
* 운영툴에서 강제 변경 가능하도록 설계

### 오프라인 보상과 길드 버프

* 길드 버프는 가입 이후 시점부터만 적용
* 탈퇴 이후 시점부터는 미적용
* 소급/잔존 적용 금지

---

## 6. 서버/클라이언트 처리 구분

## 6-1. 서버 처리

서버 권한으로 처리해야 하는 항목:

* 길드 생성/수정/해산
* 길드 가입/승인/탈퇴/추방
* 출석 가능 여부 검증 및 보상 지급
* 기부 가능 여부 검증 및 재화 차감/보상 지급
* 길드 EXP 증가 및 레벨업
* 길드 버프 적용값 산출
* 길드 상점 구매 가능 여부 검증 및 구매 처리
* 길드 총 전투력 캐시 갱신
* 주간 랭킹 정산
* 랭킹 보상 생성 및 수령 처리
* 가입 대기시간 및 보상 자격 검증

원칙:

* **재화 증감**
* **보상 지급**
* **랭킹 정산**
* **버프 적용**
  이 4가지는 전부 서버 권한으로 처리한다.

---

## 6-2. 클라이언트 처리

클라이언트는 다음 역할에 집중한다.

* 화면 렌더링
* 탭 이동 및 정렬
* 버튼 비활성화 처리
* 서버 응답 기반 연출 재생
* 남은 시간 표시
* 레드닷 표시
* 길드 정보 캐시 표시

주의:

* 클라이언트에서 출석 가능/불가를 임의 확정하지 않음
* 전투력 총합, 랭킹 순위, 버프 최종 수치는 서버값만 사용
* 구매/수령 버튼은 요청 후 응답 전까지 잠금 처리

---

## 6-3. 권장 처리 방식

* 길드 총 전투력은 실시간 합산 대신 이벤트 기반 갱신

  * 로그인
  * 성장
  * 장비 변경
  * 스킬 강화
  * 주기적 백그라운드 보정
* 랭킹 화면은 약간의 지연 허용
* 최종 주간 정산은 별도 배치 작업으로 처리

## 정리

이 길드 시스템의 핵심은 **복잡한 협동전 없이도 매일 들어오게 만들고, 성장 재화를 추가 회수하게 만드는 것**입니다.
즉, `출석 → 기부 → 버프 → 상점 → 주간 랭킹`의 짧은 루프만으로 충분히 기능하는, 아주 가벼운 길드 구조입니다.

다음 단계로 이어서 만들면 좋은 문서는 **길드 레벨 테이블**, **길드 상점 상품표**, **길드 UI 와이어프레임 텍스트**입니다.
