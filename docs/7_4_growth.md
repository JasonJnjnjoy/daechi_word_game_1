# 성장 시스템 기획서

## 1. 시스템 개요

성장 시스템은 **골드를 소비해 캐릭터의 기본 능력치를 확정적으로 강화하는 상시 성장축**이다.
이 시스템의 목적은 유저가 사냥과 오프라인 보상으로 얻은 골드를 즉시 소비해 “강해졌다”는 체감을 빠르게 얻도록 만드는 것이다.

이 게임에서 성장 시스템은 다음 역할만 담당한다.

* **성장**: 기본 능력치 강화
* **장비**: 부위별 장착과 교체, 소환 기반 성장
* **스킬(#15)**: 스킬 획득 및 스킬 레벨 강화
* **스킬 세팅(#21)**: 전투에 사용할 스킬 장착과 프리셋 관리

즉, 성장은 **가장 단순하고 확실한 골드 소모처**이며, RNG 없이 누적되는 기초 성장 수단이다.

### 핵심 정책

* 사용 재화: **골드만 사용**
* 강화 성공률: **100%**
* 실패, 파괴, 하락: **없음**
* 적용 범위: 지역 전투, 던전, 오프라인 보상 전투력, 랭킹 산정에 공통 반영
* 별도 캐릭터 경험치/레벨업 시스템: **MVP에서는 제외**
* 초기화/환불 기능: **MVP에서는 제외**

### 성장 대상 능력치

기본 구성은 아래 5종으로 제한한다.

1. 공격력
2. 체력
3. 방어력
4. 치명타 확률
5. 치명타 피해

초반 복잡도를 낮추기 위해 다음처럼 해금해도 된다.

* 공격력 / 체력 / 방어력: 초기 오픈
* 치명타 확률: 일정 지역 진입 시 오픈
* 치명타 피해: 그 다음 지역 진입 시 오픈

## 2. 규칙 설명

### 2-1. 기본 규칙

유저는 성장 화면에서 원하는 능력치를 선택하고 골드를 소비해 레벨을 올린다.
각 능력치는 서로 독립적인 레벨을 가지며, 강화 즉시 최종 전투력에 반영된다.

예시 흐름은 다음과 같다.

1. 사냥 또는 오프라인 보상으로 골드 획득
2. 성장 메뉴 진입
3. 능력치 선택
4. 1회 / 10회 / MAX 강화
5. 골드 차감
6. 능력치 상승 및 전투력 갱신

### 2-2. 강화 방식

지원하는 강화 단위는 다음 3개다.

* **1회 강화**
* **10회 강화**
* **MAX 강화**: 현재 보유 골드로 가능한 최대 횟수만큼 자동 강화

### 2-3. 비용 규칙

성장 비용은 능력치별 기본 비용과 증가 계수를 가진다.
레벨이 오를수록 비용이 점진적으로 증가한다.

예시 공식:

```text
강화비용(level) = ceil(기본비용 × 비용계수^(현재레벨))
```

일괄 강화는 다음처럼 계산한다.

```text
총비용 = 현재레벨부터 강화횟수만큼의 개별 비용 합
```

### 2-4. 능력치 상승 규칙

능력치 상승량은 마스터 데이터 기준으로 관리한다.

* 공격력 / 체력 / 방어력: 고정 수치 증가
* 치명타 확률 / 치명타 피해: 퍼센트 증가

예시:

```text
공격력 증가량 = 레벨 × level당 증가값
체력 증가량 = 레벨 × level당 증가값
치명타 확률 증가량 = 레벨 × level당 증가값(bp)
```

퍼센트 계열은 float 대신 정수 단위 관리가 좋다.

* 1% = 100bp(basis point)로 저장
* 예: 0.05% = 5bp

### 2-5. 제한 규칙

* 잠금 상태의 능력치는 강화할 수 없다.
* 최대 레벨 도달 시 더 이상 강화할 수 없다.
* 치명타 확률은 최종 합산 후 별도 상한을 둘 수 있다.
  예: 최종 치명타 확률 최대 50% 또는 60%
* 성장 수치는 영구 누적이며, 장비 교체나 스킬 변경으로 사라지지 않는다.

### 2-6. 기획 의도

성장은 유저가 막혔을 때 가장 먼저 찾는 “확정 성장”이어야 한다.
그래서 아래 요소는 의도적으로 제외한다.

* 성장 실패 확률
* 재료 합성
* 성장 트리 분기
* 무료 초기화
* 자동 분배 AI

## 3. 데이터 구조

성장 시스템은 크게 **마스터 데이터**, **유저 상태 데이터**, **로그 데이터**로 나눈다.

### 3-1. 마스터 데이터

```text
GrowthStatMaster
- stat_id
- stat_name
- stat_type            // ATK, HP, DEF, CRIT_RATE, CRIT_DMG
- unlock_type          // NONE, AREA_CLEAR, ACCOUNT_STAGE 등
- unlock_value
- max_level
- base_cost_gold
- cost_growth_rate
- value_per_level
- value_type           // FLAT, BP
- display_order
- icon_id
- is_active
```

설명:

* `unlock_type`, `unlock_value`: 언제 오픈되는지 제어
* `value_type`: 고정 수치인지 퍼센트(bp)인지 구분
* `is_active`: 운영 중 비활성화 가능 여부

### 3-2. 유저 상태 데이터

```text
UserGrowthStat
- user_id
- stat_id
- current_level
- total_spent_gold
- updated_at
```

한 유저는 능력치별로 1개의 row를 가진다.

### 3-3. 계산 결과 캐시 데이터

전투 진입, 랭킹 계산, UI 표시 최적화를 위해 최종 능력치를 캐싱할 수 있다.

```text
UserDerivedStat
- user_id
- final_atk
- final_hp
- final_def
- final_crit_rate_bp
- final_crit_dmg_bp
- combat_power
- recalculated_at
```

### 3-4. 로그 데이터

```text
GrowthUpgradeLog
- log_id
- user_id
- stat_id
- upgrade_mode         // ONE, TEN, MAX
- before_level
- after_level
- spent_gold
- result               // SUCCESS, FAIL
- fail_reason
- area_id
- created_at
```

## 4. UI 구성

성장 UI는 **직관적이고 반복 사용하기 쉬운 구조**가 중요하다.
복잡한 탭 분리 없이 한 화면에서 대부분의 정보를 처리한다.

### 4-1. 화면 구성안

상단에는 현재 상태 요약을 배치한다.

* 현재 전투력
* 현재 골드 보유량
* 도움말 버튼
* 뒤로가기

중앙에는 성장 능력치 리스트를 표시한다.
각 항목에는 아래 정보가 들어간다.

* 능력치 아이콘
* 능력치 이름
* 현재 레벨
* 현재 적용 수치
* 다음 레벨 상승 수치
* 1회 강화 비용
* 잠금 상태라면 해금 조건 문구

하단에는 공통 강화 조작 영역을 둔다.

* 1회 / 10회 / MAX 토글
* 강화 버튼
* 길게 누르기 연속 강화(선택)
* 강화 성공 연출, 전투력 상승 숫자 연출

### 4-2. UX 정책

* 강화 가능한 항목은 버튼 활성화
* 골드 부족 시 버튼 비활성화 또는 클릭 시 토스트 노출
* MAX 강화 시 “예상 상승 결과”를 미리 보여줄 수 있음
* 잠긴 능력치는 회색 처리 + 해금 조건 표시
* 성장 가능 항목이 1개 이상이면 메인 메뉴에 레드닷 표시

### 4-3. UI 문구 예시

* 골드가 부족합니다.
* 최대 레벨에 도달했습니다.
* 지역 5 도달 시 해금됩니다.
* 공격력 +15 상승
* 10회 강화 완료

## 5. 예외 처리

성장 시스템은 반복 클릭이 많기 때문에 예외 처리 기준이 명확해야 한다.

### 5-1. 골드 부족

상황: 클라이언트에서 강화 가능해 보였지만 서버 처리 시 골드가 부족한 경우
처리:

* 서버는 강화 실패 반환
* 클라이언트는 최신 골드값 동기화
* “골드가 부족합니다” 토스트 출력

### 5-2. 중복 요청

상황: 유저가 연속 탭으로 강화 요청을 여러 번 보냄
처리:

* 서버에서 사용자 단위 락 또는 순차 처리
* 중복 요청은 무시하거나 뒤 요청 실패 처리
* 클라이언트는 처리 중 버튼 잠금

### 5-3. 잠금 능력치 요청

상황: 해금 전 능력치를 비정상 요청
처리:

* 서버는 무조건 차단
* 실패 사유 코드 반환
* 클라이언트는 잠금 상태 유지

### 5-4. 최대 레벨 초과 요청

상황: MAX 강화 또는 연속 강화 도중 최대 레벨 도달
처리:

* 가능한 구간까지만 성공 처리
* 도달 시점 이후는 무시
* 최종 레벨과 실제 소모 골드만 반환

### 5-5. 배치 강화 중 부분 성공

상황: 10회 요청했지만 실제로는 6회만 가능한 경우
처리:

* 서버는 **가능한 횟수까지만 성공 처리**
* 실제 강화 횟수, 실제 소모 골드, 최종 레벨 반환
* 클라이언트는 “6회 강화 완료”처럼 실제 결과 표시

### 5-6. 통신 실패

상황: 요청 후 응답 전에 네트워크가 끊김
처리:

* 재접속 시 서버 기준 상태 재동기화
* 클라이언트 로컬 상태를 신뢰하지 않음
* 중복 지급/중복 강화가 발생하지 않도록 서버 원장 기준 유지

## 6. 서버/클라이언트 처리 구분

### 6-1. 서버 처리

서버는 성장 관련 핵심 상태를 전부 권한 있게 관리한다.

서버 책임:

* 능력치 해금 여부 판정
* 현재 골드 검증
* 강화 가능 횟수 계산
* 골드 차감
* 성장 레벨 반영
* 최종 능력치 재계산
* 전투력 갱신
* 로그 저장
* 랭킹 반영용 데이터 갱신

즉, **강화 성공 여부와 최종 결과는 반드시 서버 기준**이다.

### 6-2. 클라이언트 처리

클라이언트는 주로 표현과 편의 기능을 담당한다.

클라이언트 책임:

* UI 렌더링
* 강화 전 예상 비용/예상 상승량 표시
* 버튼 활성/비활성 처리
* 연출, 이펙트, 토스트
* 도움말/가이드 문구 표시
* 레드닷 표시

### 6-3. 권장 처리 흐름

성장 요청 시 권장 순서는 다음과 같다.

1. 클라이언트가 강화 요청 전송
2. 서버가 잠금/최대레벨/골드 검증
3. 서버가 실제 강화 횟수 계산
4. 서버가 골드 차감 + 레벨 갱신을 원자적으로 처리
5. 서버가 최종 능력치 재계산
6. 서버가 결과 응답
7. 클라이언트가 UI 갱신 및 연출 재생

### 6-4. 주의점

* 오프라인 보상 수령 직후 성장 진입 시 최신 골드 기준으로 동기화 필요
* 성장, 상점 구매, 소환이 모두 골드를 쓸 수 있다면 동시 처리 충돌 방지 필요
* 전투 중에도 성장 수치를 즉시 반영할지, 다음 전투부터 반영할지 정책을 정해야 한다.
  **MVP에서는 즉시 반영하되, 현재 진행 전투는 유지하고 다음 전투 웨이브부터 적용**이 안정적이다.

## 7. 로그 및 운영 포인트

성장 시스템은 게임의 핵심 골드 소모처이므로, 운영에서는 단순 사용량보다 **막힘 구간과 소모 효율**을 보는 것이 중요하다.

### 7-1. 필수 로그

최소 아래 이벤트는 남겨야 한다.

* 성장 화면 진입
* 성장 강화 시도
* 성장 강화 성공
* 성장 강화 실패
* MAX 강화 사용
* 잠금 능력치 클릭
* 골드 부족 상태에서 강화 클릭

### 7-2. 핵심 운영 지표

운영에서 반드시 봐야 할 값은 아래다.

* 일자별 성장 시스템 골드 소모량
* 능력치별 투자 비중
* 지역/스테이지 구간별 평균 성장 레벨
* 특정 구간에서 성장 정체가 발생하는지 여부
* 성장 메뉴 진입 대비 실제 강화 전환율
* 성장 레드닷 노출 대비 클릭률
* 성장 투자 후 다음날 잔존율 변화

### 7-3. 밸런스 조정 포인트

운영 중 조정 가능한 핵심 값은 아래다.

* 능력치별 기본 비용
* 비용 증가 계수
* 능력치별 레벨당 상승값
* 능력치 해금 시점
* 최대 레벨
* 치명타 확률 상한

예를 들어 이런 판단이 가능해야 한다.

* 초반에 유저가 공격력만 올리고 체력/방어력을 무시하는가
* 중반 이후 치명타 계열이 과도하게 효율적인가
* 성장 비용이 너무 가팔라서 장비/소환보다 재미가 떨어지는가
* 반대로 성장 효율이 너무 좋아 장비/스킬 존재감이 약해지는가

### 7-4. 어뷰징 및 장애 대응 포인트

* 비정상적으로 짧은 시간에 과도한 성장 요청 발생 여부
* 골드 보유량 대비 불가능한 강화 결과 발생 여부
* 재접속 반복으로 중복 강화가 발생하는지 여부
* 업데이트 후 특정 능력치가 비정상적으로 많이 선택되는지 여부

# 정리

이 설계에서 성장 시스템은 한 줄로 정리하면 다음이다.

**“골드를 써서 기본 능력치를 확정적으로 올리는 가장 직관적인 상시 성장 시스템”**

이 방향으로 가면 장비/스킬과 역할이 깔끔하게 분리되고, 유저도 사냥 → 골드 획득 → 즉시 성장이라는 흐름을 쉽게 이해할 수 있다.

원하면 다음 단계로 이어서 **성장 수치 예시 밸런스 테이블(초반/중반/후반)**까지 바로 작성해드릴게.
