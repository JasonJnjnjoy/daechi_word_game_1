# 재화 시스템 기획서

본 문서는 방치형 게임의 **재화 시스템**을 정의한다.
설계 목표는 **단순성, 역할 분리, 성장 동기 유지, 운영 편의성**이다.

---

## 1. 시스템 개요

재화 시스템은 유저의 성장 속도, 플레이 동기, 반복 플레이 유도, 과금 동선, 길드 참여 동선을 연결하는 핵심 시스템이다.
본 프로젝트에서는 재화를 6종으로 제한하여 복잡도를 낮추고, 각 재화의 역할이 겹치지 않도록 설계한다.

### 1-1. 재화 목록

| 재화     | 분류       | 핵심 역할        | 주요 획득처                   | 주요 사용처                 |
| ------ | -------- | ------------ | ------------------------ | ---------------------- |
| 골드     | 기본 성장 재화 | 일상 성장의 중심    | 몬스터 처치, 오프라인 보상, 퀘스트, 던전 | 성장, 장비 강화, 스킬 강화       |
| 다이아    | 프리미엄 재화  | 선택적 가속/구매    | 업적, 이벤트, 시즌패스, 상점 구매, 우편 | 상점 상품, 티켓 번들, 던전 티켓 보충 |
| 장비 소환권 | 전용 티켓    | 장비 획득        | 퀘스트, 이벤트, 시즌패스, 상점       | 장비 소환                  |
| 스킬 소환권 | 전용 티켓    | 스킬 획득        | 퀘스트, 이벤트, 시즌패스, 상점       | 스킬 소환                  |
| 던전 티켓  | 입장 재화    | 일일/반복 콘텐츠 제어 | 자연 회복, 이벤트, 상점, 우편       | 던전 입장                  |
| 길드 코인  | 협동 재화    | 길드 참여 보상     | 길드 출석, 길드 미션, 길드 보상      | 길드 상점                  |

### 1-2. 설계 원칙

1. 모든 재화는 **계정 단위**로 관리한다.
2. 모든 재화 수량은 **정수형**으로 처리한다. 소수점은 사용하지 않는다.
3. 유저 간 재화 거래, 선물, 교환은 지원하지 않는다.
4. 재화 간 직접 환전은 지원하지 않는다. 단, 상점 상품 형태의 간접 교환만 허용한다.
5. 다이아는 **유료/무료 구분 없이 단일 재화**로 운영한다.
6. 장비/스킬 소환은 각각 전용 티켓만 사용하며, **소환 화면에서 다이아 직접 소모는 지원하지 않는다**.
7. 던전 티켓만 자연 회복 기능을 가진다.
8. 오프라인 보상은 최대 12시간이며, 핵심 지급 재화는 골드다.

## 2. 규칙 설명

## 2-1. 공통 규칙

### 재화 보유 규칙

* 골드, 다이아, 길드 코인은 사실상 상한이 없는 구조로 운영하되, 서버 내부적으로는 `BIGINT` 범위를 사용한다.
* 장비 소환권, 스킬 소환권도 일반적으로 상한 없이 누적 가능하게 운영한다.
* 던전 티켓은 **자연 회복 상한**과 **절대 보유 상한**을 분리한다.

### 던전 티켓 규칙

* 자연 회복: **1시간당 1개**
* 자연 회복 상한: **10개**
* 절대 보유 상한: **99개**
* 이벤트/우편/상점으로 지급받은 티켓은 10개를 초과해도 보유 가능하나, 99개 초과분은 우편으로 전환하거나 지급 실패 처리 로그를 남긴다.

### 일일 리셋 규칙

* 일일 획득 제한, 일일 구매 제한, 일일 미션 보상 횟수는 서버 기준 일일 리셋 시각에 초기화된다.
* 재화 잔액 자체는 리셋되지 않는다.

---

## 2-2. 재화별 획득 및 소비 규칙

### 골드

골드는 가장 기본적인 성장 재화이며, 유저가 가장 자주 획득하고 소비하는 재화다.

**획득처**

* 지역 몬스터 처치
* 오프라인 보상
* 퀘스트 보상
* 던전 클리어 보상
* 이벤트/시즌패스 보상
* 우편 보상

**사용처**

* 성장 스탯 레벨업
* 장비 강화 비용
* 스킬 강화 비용
* 일부 시스템 해금 비용(필요 시)

**획득 공식**

* 실시간 골드 획득량 = `현재 지역 몬스터 평균 골드 × 분당 처치량`
* 오프라인 골드 = `현재 기준 파밍 지역 분당 골드 × 누적 시간(최대 720분)`

**운영 의도**

* 골드는 항상 들어오고 항상 빠져나가는 구조여야 한다.
* 장기적으로 남는 재화가 되지 않도록 강화 비용 곡선과 함께 관리한다.

---

### 다이아

다이아는 프리미엄 재화이며, 일반 반복 플레이로 대량 획득되면 안 된다.

**획득처**

* 업적
* 이벤트
* 시즌패스
* 우편 보상
* 상점 구매

**사용처**

* 상점 내 패키지 구매
* 장비/스킬 소환권 번들 구매
* 던전 티켓 즉시 보충
* 한정 이벤트 상품 구매

**규칙**

* 일반 지역 사냥 및 오프라인 보상에서는 지급하지 않는다.
* 성장의 직접 재화라기보다 **선택적 가속/편의 재화**로 사용한다.
* 다이아는 부족한 티켓을 소환 화면에서 자동 대체하지 않는다. 반드시 상점 구매를 거치게 한다.

---

### 장비 소환권

장비 획득 전용 재화다.

**획득처**

* 퀘스트
* 이벤트
* 시즌패스
* 상점
* 우편

**사용처**

* 장비 소환 1회 / 10회

**규칙**

* 장비 소환에만 사용된다.
* 골드/다이아로 직접 대체되지 않는다.
* 장비 소환권 부족 시 소환 버튼은 비활성화되거나, 상점 이동 팝업을 노출한다.

---

### 스킬 소환권

스킬 획득 전용 재화다.

**획득처**

* 퀘스트
* 이벤트
* 시즌패스
* 상점
* 우편

**사용처**

* 스킬 소환 1회 / 10회

**규칙**

* 스킬 소환에만 사용된다.
* 장비 소환권과 절대 혼용되지 않는다.
* 스킬 소환권 부족 시 상점 또는 획득처 안내 팝업을 제공한다.

---

### 던전 티켓

던전 플레이 횟수를 통제하는 시간형 재화다.

**획득처**

* 자연 회복
* 이벤트
* 상점
* 우편
* 특정 일일 보상

**사용처**

* 던전 입장

**규칙**

* 입장 시 서버에서 차감 후 던전 생성/입장 처리한다.
* 입장 실패 시 차감 복구 정책이 필요하다.
* 자연 회복은 서버 시간을 기준으로만 계산한다.
* 클라이언트 로컬 시간은 신뢰하지 않는다.

---

### 길드 코인

길드 참여에 대한 보상 재화다.

**획득처**

* 길드 출석
* 길드 일일 미션
* 길드 누적 기여 보상
* 길드 관련 이벤트 보상

**사용처**

* 길드 상점

**규칙**

* 길드 탈퇴 시 길드 코인은 유지한다.
* 단, 길드 상점 이용은 길드 가입 상태에서만 가능하다.
* 일반 상점이나 성장 시스템에서는 사용하지 않는다.

---

## 2-3. 재화 부족 처리 규칙

1. 재화가 부족한 경우 서버는 요청을 거절하고 최신 잔액을 반환한다.
2. 클라이언트는 부족 팝업을 띄우고, 해당 재화의 대표 획득처로 이동 버튼을 제공한다.
3. 배치 소비(예: 10회 소환)는 **필요 수량 전부 충족 시에만 가능**하다.
4. 부분 수행은 허용하지 않는다. 예를 들어 소환권 7장 보유 시 10회 소환 요청은 실패 처리한다.

## 3-4. 공용 보상 구조

다른 시스템과 연동하기 쉽도록 보상은 공용 포맷으로 관리한다.

```json
[
  { "type": "gold", "amount": 1500 },
  { "type": "equip_ticket", "amount": 2 }
]
```

퀘스트, 이벤트, 시즌패스, 우편은 모두 이 구조를 사용해 재화 지급을 요청한다.

## 4. UI 구성

UI는 “항상 필요한 것만 보이게” 설계한다.
6종 재화를 한 화면에 모두 고정 노출하지 않는다.

## 4-1. 상단 고정 UI

상단 공통 바에는 다음 2종만 상시 노출한다.

* 골드
* 다이아

이 두 재화는 대부분의 화면에서 공통적으로 의미가 있으므로 고정 노출한다.

---

## 4-2. 기능별 재화 노출

### 소환 화면

* 장비 소환 화면: 장비 소환권 표시
* 스킬 소환 화면: 스킬 소환권 표시

### 던전 화면

* 던전 티켓 현재 수량
* 다음 회복까지 남은 시간 표시

### 길드 화면

* 길드 코인 현재 수량 표시

---

## 4-3. 재화 상세 팝업

재화 아이콘 클릭 시 상세 팝업을 제공한다.

표시 내용:

* 현재 보유량
* 주요 획득처 2~3개
* 주요 사용처 2~3개
* 부족 시 이동 가능한 대표 화면 버튼

예시:

* 골드 부족 → 지역/오프라인 보상 안내
* 장비 소환권 부족 → 상점/이벤트 이동
* 던전 티켓 부족 → 회복 시간 안내 + 상점 이동

---

## 4-4. 보상 팝업

재화 획득 시에는 공통 보상 팝업을 사용한다.

규칙:

* 같은 종류 재화는 합산 표시
* 획득량은 `+1,500`처럼 직관적으로 노출
* 희소 재화(다이아, 각종 티켓)는 골드보다 시각적으로 강조
* 연속 획득 시 팝업이 과도하게 누적되지 않도록 묶음 처리

---

## 4-5. 부족 팝업

재화 부족 팝업에는 다음 요소를 넣는다.

* 부족한 재화명
* 현재 보유량 / 필요 수량
* 대표 획득처 버튼
* 취소 버튼

예시:

* “장비 소환권이 부족합니다. 상점으로 이동하시겠습니까?”

---

## 5. 예외 처리

| 상황                  | 처리 방식                     |
| ------------------- | ------------------------- |
| 재화 부족 요청            | 서버에서 거절, 최신 잔액 반환         |
| 중복 클릭/중복 패킷         | `trace_id` 기준으로 1회만 처리    |
| 재화 오버플로우            | 정책에 따라 허용 / 우편 전환 / 거절 처리 |
| 입장 요청 후 던전 생성 실패    | 티켓 차감 롤백                  |
| 오프라인 보상 중복 수령 시도    | 서버에서 1회만 지급               |
| 클라이언트 시간 조작         | 무시, 서버 시간 기준 재계산          |
| 일일 리셋 직전/직후 요청 경합   | 서버 시각 기준으로 한 번만 판정        |
| 길드 탈퇴 상태에서 길드 상점 접근 | 상점 접근 차단                  |
| 네트워크 끊김 후 재접속       | 서버 잔액 재동기화 후 UI 갱신        |

### 추가 처리 원칙

* 잔액이 음수가 되는 상황은 절대 허용하지 않는다.
* 모든 재화 변경은 원자적으로 처리한다.
* 하나의 요청에서 여러 재화가 동시에 변할 경우, 전부 성공 또는 전부 실패해야 한다.

## 6. 서버/클라이언트 처리 구분

| 항목             | 서버     | 클라이언트 |
| -------------- | ------ | ----- |
| 실제 재화 잔액 보관    | O      | X     |
| 재화 지급/차감 판정    | O      | X     |
| 오프라인 보상 계산     | O      | X     |
| 던전 티켓 자연 회복 계산 | O      | X     |
| 상점 구매 검증       | O      | X     |
| 부족 팝업 표시       | X      | O     |
| 재화 애니메이션/연출    | X      | O     |
| 임시 UI 카운트다운 표시 | 기준값 제공 | O     |
| 거래 로그 기록       | O      | X     |
| 화면별 노출 순서/아이콘  | X      | O     |

### 처리 원칙

* 클라이언트는 재화를 “보여주는 역할”만 한다.
* 서버는 재화를 “판정하고 확정하는 역할”을 한다.
* 클라이언트에서 보이는 카운트다운은 연출용이며, 최종 수량은 서버 응답으로 갱신한다.

### 권장 처리 플로우

1. 클라이언트가 소비 요청 전송
2. 서버가 잔액/한도/조건 검증
3. 서버가 `user_wallet` 갱신
4. 서버가 `currency_tx_log` 기록
5. 서버가 결과와 변경 재화 목록 반환
6. 클라이언트가 UI 반영 및 연출 재생

## 7. 로그 및 운영 포인트

## 7-1. 필수 로그

모든 재화 증감은 반드시 로그로 남긴다.

최소 기록 항목:

* 누가 받았는지
* 어떤 재화인지
* 얼마나 변했는지
* 왜 변했는지
* 어떤 시스템에서 발생했는지
* 처리 전/후 잔액
* 중복 요청 식별값

---

## 7-2. 핵심 운영 지표

### 골드

* 일일 총 생산량
* 일일 총 소모량
* 순증가량
* 유저 평균 보유량
* 구간별 골드 부족률

### 다이아

* 무료 지급량 / 구매량
* 사용처별 소모 비중
* 상점 상품 전환율
* 부족 팝업 발생률

### 장비/스킬 소환권

* 일일 공급량
* 소환 사용량
* 보유 누적 분포
* 10회 소환 진입률

### 던전 티켓

* 자연 회복량
* 실제 소비량
* 회복 상한 초과 방치율
* 상점 보충 구매율

### 길드 코인

* 길드 활동자 평균 획득량
* 길드 상점 소비율
* 길드 코인 누적 방치율

---

## 7-3. 운영 조정 포인트

운영팀이 가장 자주 조정하게 될 포인트는 다음과 같다.

1. **골드 드롭 배율**

   * 지역별 골드 인플레이션 조정

2. **오프라인 보상 효율**

   * 진도 정체 구간 완화용

3. **티켓 공급량**

   * 이벤트/시즌패스/상점에서 조정

4. **던전 티켓 회복 속도**

   * 일일 접속 빈도 조절용

5. **길드 코인 획득량 및 길드 상점 가격**

   * 길드 참여 유도 강도 조절용

6. **다이아 상품 가격**

   * 과금 효율과 무과금 진입감을 동시에 조정

---

## 7-4. 운영상 주의 사항

* 골드는 너무 많이 풀리면 성장 압축이 발생하고, 너무 적으면 게임이 답답해진다.
* 다이아를 일반 플레이에서 과하게 공급하면 상점 가치가 붕괴한다.
* 장비/스킬 소환권을 지나치게 적게 주면 소환의 재미가 사라지고, 너무 많이 주면 성장 곡선이 무너진다.
* 던전 티켓 자연 회복 상한이 너무 낮으면 접속 압박이 심해지고, 너무 높으면 일일 루틴이 느슨해진다.
* 길드 코인이 쌓이기만 하고 쓰이지 않으면 길드 참여 동기가 약해진다.
