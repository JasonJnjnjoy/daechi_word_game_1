좋아. 아래는 **MVP 기준의 ‘랭킹’ 시스템 기획서**다.
전제는 네가 준 조건을 그대로 따르고, **직접 PvP 없이 비교/보상 중심**, **최대한 단순한 운영**에 맞췄다.

---

# 랭킹 시스템 기획서

## 0. 기능 정의

랭킹은 유저 간 직접 전투 없이, **성장 정도와 진행 속도를 비교**하고 **주간 보상을 지급**하는 시스템이다.
이 시스템은 다음 3종의 보드를 제공한다.

* **개인 지역 랭킹**: 누가 가장 높은 지역까지 도달했는지
* **개인 전투력 랭킹**: 누가 가장 높은 전투력을 기록했는지
* **길드 랭킹**: 어느 길드가 주간 기준 가장 높은 길드 점수를 획득했는지

### 설계 방향

* 랭킹의 본질은 **명예 + 주간 보상**
* **직접 전투, 복수전, 방어덱, 공격기록 없음**
* 보상은 기존 재화만 사용
* 지나친 보상 인플레이션 방지를 위해 **개인 주간 핵심 보상은 지역 랭킹 중심**
* 길드전이 없으므로 길드 랭킹은 **길드 활동 점수 기반**

### 기본 운영 단위

* **서버군 단위 랭킹** 권장

  * 이유: 신서버/구서버 격차 완화
  * 서버가 1개뿐이면 전체 통합 랭킹으로 동일 룰 적용

## 1. 시스템 개요

랭킹 시스템의 목적은 다음과 같다.

1. **중장기 동기 부여**

   * 지역을 더 밀고, 전투력을 더 올릴 명확한 목표를 제공

2. **과금/무과금 모두의 경쟁 포인트 제공**

   * 전투력만이 아니라 **지역 진척도** 자체를 주 경쟁축으로 사용

3. **주간 보상 회수처 제공**

   * 시즌패스, 이벤트, 길드와 함께 주간 반복 루프의 보상 회수처 역할 수행

4. **직접 PvP 대체**

   * 전투 밸런싱 없이도 경쟁 체감을 줄 수 있음

### 랭킹 구성 요약

| 랭킹 종류     | 대상 | 기준            | 보상 여부         | 비고       |
| --------- | -- | ------------- | ------------- | -------- |
| 개인 지역 랭킹  | 유저 | 최고 도달 지역/스테이지 | O             | 핵심 경쟁 보드 |
| 개인 전투력 랭킹 | 유저 | 최고 기록 전투력     | 선택적/소액 또는 명예형 | 참고 보드 성격 |
| 길드 랭킹     | 길드 | 주간 길드 점수      | O             | 길드 코인 지급 |

## 2. 규칙 설명

## 2-1. 참가 조건

### 개인 랭킹 참가 조건

* **메인 지역 일정 구간 도달 시 오픈**
* 권장 조건: **3지역 진입** 또는 **지역 1 클리어**
* 조건 미달 시 랭킹 메뉴는 보이되, “특정 지역 도달 시 오픈” 문구 출력

### 길드 랭킹 참가 조건

* 길드 가입 후 즉시 목록은 볼 수 있음
* 다만 **길드 보상 수령은 정산 시점 기준 가입 상태 + 최소 가입 유지 시간 충족** 시 가능
* 권장 기준: **정산 시점 기준 24시간 이상 가입**

---

## 2-2. 랭킹 종류별 점수 규칙

### A. 개인 지역 랭킹

유저의 **최고 도달 지역/스테이지**를 기준으로 정렬한다.

#### 점수 산식 예시

```text
지역 점수 = (최고 지역 ID × 1,000,000) + (최고 스테이지 ID × 1,000) + 세부 진행값
```

세부 진행값은 선택 사항이다.

* 사용하는 경우: 동일 스테이지 내 처치 진행률, 웨이브 진행도 등
* 사용하지 않는 경우: 지역 + 스테이지만으로 정렬

#### 정렬 우선순위

1. 지역 점수 높은 순
2. 같은 점수면 **해당 점수 최초 달성 시각이 빠른 순**
3. 그래도 같으면 **계정 생성 시각이 빠른 순**

> 지역 랭킹은 이 게임의 핵심 랭킹이다.
> 이유는 코어 루프가 “지역 진행” 중심이기 때문이다.

---

### B. 개인 전투력 랭킹

유저가 기록한 **최고 전투력(peak battle power)** 기준으로 정렬한다.

#### 점수 규칙

* 현재 전투력이 아니라 **최고 기록 전투력** 사용
* 장비 해제/교체 등으로 현재 전투력이 내려가도 랭킹 점수는 유지
* 전투력은 서버가 계산한 값만 인정

#### 정렬 우선순위

1. 최고 전투력 높은 순
2. 같은 점수면 해당 점수 최초 달성 시각이 빠른 순
3. 그래도 같으면 계정 생성 시각이 빠른 순

#### 운영 의도

* 전투력 랭킹은 성장 자랑용/참고용 성격이 강함
* 과금 편차가 크게 작용할 수 있으므로, **지역 랭킹 대비 보상 비중은 낮게 운영**하는 것이 안정적

---

### C. 길드 랭킹

길드의 **주간 길드 점수** 기준으로 정렬한다.

#### 점수 원칙

길드 점수는 길드 시스템에서 계산한 주간 활동 점수를 사용한다.

예:

* 길드 출석
* 길드 기부
* 길드 콘텐츠 참여
* 길드원 활동 누적치

#### 랭킹 시스템 관점 정의

랭킹 시스템은 길드 시스템이 넘겨준 아래 값을 사용한다.

```text
주간 길드 점수 = 해당 주간에 누적된 길드 활동 점수
```

#### 정렬 우선순위

1. 주간 길드 점수 높은 순
2. 같은 점수면 해당 점수 최초 달성 시각이 빠른 순
3. 그래도 같으면 길드 생성 시각이 빠른 순

---

## 2-3. 집계 및 갱신 규칙

### 점수 갱신 트리거

#### 개인 지역 랭킹

* 지역 클리어
* 최고 스테이지 갱신
* 세부 진행 수치 갱신 시(사용하는 경우)

#### 개인 전투력 랭킹

* 성장 수치 강화
* 장비 강화/교체
* 스킬 획득/강화
* 전투력 재계산 발생 시

#### 길드 랭킹

* 길드 시스템에서 길드 점수 변경 이벤트 발생 시

### 갱신 방식

* **실제 점수는 서버에서 즉시 갱신**
* **랭킹 보드 표시 순위는 캐시 기준 주기 갱신**
* 권장 보드 반영 주기: **1~5분**

이렇게 하면:

* 서버 부하 감소
* 유저는 충분히 “거의 실시간”처럼 느낄 수 있음

---

## 2-4. 정산 주기 및 보상 규칙

### 정산 주기

* **주간 정산** 권장
* 예: 매주 월요일 00:00

### 보상 지급 방식

* 정산 완료 후 **우편으로 자동 지급**
* 별도 수동 수령 UI를 만들지 않아도 됨
* 우편 만료 기간 내 미수령 시 삭제

### 보상 구성 원칙

#### 개인 지역 랭킹

* 다이아
* 장비 소환권
* 스킬 소환권

#### 개인 전투력 랭킹

* 무보상 또는 소액 보상
* 또는 명예형(프레임/칭호)으로 추후 확장 가능

#### 길드 랭킹

* 길드 코인 중심
* 정산 시점 길드 소속 조건을 만족한 길드원에게 지급

### 보상 중복 규칙 권장안

운영 단순화를 위해 아래 둘 중 하나를 추천한다.

#### 안 A. 완전 중복 지급

* 지역/전투력/길드 보상 모두 각각 지급
* 장점: 단순
* 단점: 보상 총량 증가

#### 안 B. 개인 보상 1종 우선 지급 + 길드 별도 지급 **(권장)**

* 개인은 지역 랭킹 보상만 핵심 지급
* 전투력 랭킹은 참고/명예용
* 길드 랭킹은 별도 지급
* 장점: 경제 관리 쉬움

이 문서에서는 **안 B를 권장**한다.

### 3-6. 권장 보조 캐시

실시간 조회 부하를 줄이기 위해 Redis 등 캐시 사용 권장

* `ranking:region:{serverGroupId}`
* `ranking:power:{serverGroupId}`
* `ranking:guild:{serverGroupId}`

캐시에는 최소한 아래 정보를 보관한다.

* rank
* target_id
* display_name
* score
* auxiliary_text(지역명, 전투력, 길드명 등)

## 4. UI 구성

## 4-1. 진입 구조

* 메인 UI에서 **랭킹 버튼(#35)** 선택
* 진입 시 기본 탭은 **개인 지역 랭킹**

---

## 4-2. 화면 구성

### 상단 영역

* 탭 버튼

  * 개인 지역
  * 개인 전투력
  * 길드
* 정산까지 남은 시간
* 보상 미리보기 버튼
* 도움말 버튼

### 중앙 영역

* TOP 3 강조 영역
* 4위 이하 리스트

### 하단 고정 영역

* 내 순위
* 내 현재 기록
* 갱신 기준 안내 문구

## 4-3. 리스트 셀 구성

### 개인 랭킹 셀

* 순위
* 프로필 아이콘
* 닉네임
* 소속 길드명
* 현재 기록

  * 전투력 탭: 최고 전투력

### 길드 랭킹 셀

* 순위
* 길드 마크
* 길드명
* 길드 점수
* 길드원 수

## 4-4. 내 순위 패널

TOP 100 밖이어도 항상 보여준다.

표시 정보:

* 내 순위
* 내 점수
* 다음 순위와의 차이
* 보상 구간 정보

예:

* “현재 127위”
* “100위까지 지역 2개 차이”

## 4-5. 상태별 UI

### 참가 조건 미달

* 리스트 대신 잠금 문구 표시

* 일시적으로 보드 갱신 정지

그래도 같으면 계정/길드 생성 시각이 빠른 쪽을 우선한다.

---

## 5-2. 장비 해제 등으로 전투력 하락

전투력 랭킹은 **최고 기록 전투력 기준**이므로, 현재 전투력이 하락해도 순위가 즉시 떨어지지 않는다.
단, 운영상 악용이 확인되면 서버 재계산으로 보정할 수 있다.

---

## 5-3. 길드 탈퇴/강퇴/가입 직전 정산

길드 보상은 **정산 시점 기준 소속 + 최소 가입 유지 시간 충족** 조건으로 지급한다.

예:

* 정산 직전 가입: 보상 미지급
* 정산 직전 탈퇴/강퇴: 보상 미지급
* 길드 자체 순위는 정산 시점 길드 상태 기준 적용

---

## 5-4. 길드 해산

정산 전에 길드가 해산되면 해당 길드는 랭킹 대상에서 제외한다.
필요 시 운영툴에서 수동 복구 가능하도록 한다.

---

## 5-5. 비정상 점수 상승

다음 경우 서버에서 자동 검증 또는 운영 검토 대상 처리한다.

* 비정상적으로 빠른 지역 돌파
* 허용 불가능한 전투력 급등
* 롤백된 강화 내역이 랭킹에 남아있는 경우

이 경우:

* 점수 롤백
* 랭킹 숨김 처리
* 정산 제외 처리 가능

---

## 5-6. 닉네임 변경

랭킹 표시는 최신 닉네임을 사용한다.
단, 정산 이력은 당시 기준 표시명을 별도 저장해두면 CS 대응이 쉬워진다.

---

## 5-7. 정산 실패 또는 중복 지급

정산은 **idempotent**하게 설계해야 한다.

즉:

* 같은 회차 정산을 여러 번 실행해도
* 보상은 한 번만 지급되도록 해야 한다.

`season_id + ranking_type + target_id` 기준 중복 방지 권장.

---

## 5-8. 휴면/제재 계정

* 제재 계정은 랭킹 노출 제외
* 휴면 계정은 점수 유지 가능하지만, 운영 정책에 따라 장기 미접속 시 노출 제외 가능

---

## 6. 서버/클라이언트 처리 구분

## 6-1. 서버 처리

서버는 다음을 담당한다.

### 점수 계산

* 지역 점수 계산
* 최고 전투력 계산
* 길드 점수 반영

### 참가 조건 판정

* 랭킹 오픈 여부
* 길드 보상 지급 자격 여부

### 정렬 및 스냅샷 생성

* 순위 산정
* 동점 처리
* 정산 시점 스냅샷 고정

### 보상 지급

* 우편 발송
* 중복 지급 방지

### 비정상 탐지

* 점수 이상치 감지
* 운영 숨김 처리

### 캐시 관리

* 랭킹 보드 캐시 생성/갱신

> 핵심 원칙:
> **랭킹 관련 수치는 전부 서버 권한**으로 처리한다.

---

## 6-2. 클라이언트 처리

클라이언트는 다음만 담당한다.

* 랭킹 UI 표시
* 탭 전환
* 내 순위/보상 구간 표시
* 도움말/보상 팝업 노출
* 주기적 재조회 또는 화면 진입 시 조회

클라이언트는 점수 계산을 확정하지 않는다.
전투력 미리보기는 가능하지만, 랭킹 반영 기준값은 항상 서버값을 사용한다.

---

## 6-3. 권장 통신 방식

### 조회 API

* 랭킹 리스트 조회
* 내 순위 조회
* 보상 미리보기 조회

### 갱신 방식

* 화면 진입 시 조회
* 탭 전환 시 조회
* 백그라운드에서는 30~60초 간격 재요청 또는 수동 새로고침

실시간 푸시까지는 필요 없다.
MVP에서는 **폴링 기반**이 가장 단순하다.

---

* 실패 사유

* ranking_type
* target_id

* 우편 발송 결과

---

### 운영 제재 로그

---

## 7-2. 운영 포인트

운영툴에서 아래를 수정 가능해야 한다.

* 순위 구간별 보상
* 지급 여부 ON/OFF

---

### 랭킹 ON/OFF

* 서버군 단위로 노출/비노출 가능
* 신규 서버 안정화 기간에는 임시 비활성 가능

---

### 수동 재정산

필요 시 운영자가 특정 회차만 재정산할 수 있어야 한다.

예:

* 정산 배치 실패
* 특정 제재 계정 제외 후 재정산
* 보상 누락 복구

---

### 이상치 모니터링

* 길드 점수 집중도

* 주간 랭킹 참여 유저 수
* 내 순위 패널 조회율
* 정산 후 다음날 잔존율
* 길드 가입 전환율

# 최종 권장안 요약

이 게임에서는 랭킹을 아래처럼 두는 것이 가장 깔끔하다.

### 핵심 구조

* **개인 지역 랭킹**: 메인 경쟁 보드, 주간 핵심 보상 지급
* **개인 전투력 랭킹**: 명예/참고 보드, 보상은 축소 또는 없음
* **길드 랭킹**: 주간 길드 점수 보드, 길드 코인 지급

### 운영 방식

* 서버군 단위
* 주간 정산
* 우편 자동 지급
* 서버 권한 점수 계산
* 캐시 기반 보드 표시
* 동점은 최초 달성 시각 우선

### 단순화 포인트

* 직접 PvP 없음
* 전투 로그 없음
* 리플레이 없음
* 실시간 푸시 없음
* 개인 보상 중복 최소화

원하면 다음으로 이어서 **‘길드’ 시스템 기획서**나, **이 랭킹 시스템의 실제 UI 와이어프레임 텍스트 버전**까지 바로 작성해주겠다.
