# 지역 선택 시스템 기획서

## 1. 시스템 개요

### 1-1. 목적

지역 선택 시스템은 플레이어가 현재 **전투/파밍할 지역**을 선택하고, **다음 지역 도전**을 통해 성장 체감을 얻도록 만드는 핵심 진행 시스템이다.
이 시스템은 전투, 오프라인 보상, 성장, 퀘스트, 시즌패스, 이벤트, 랭킹과 연결된다.

### 1-2. 설계 방향

* **복잡한 월드맵 대신 단순한 선형 구조**를 사용한다.
* 플레이어는 **이미 클리어한 지역은 자유롭게 재진입**할 수 있다.
* 플레이어는 **최고 클리어 지역 + 1**까지만 도전할 수 있다.
* 지역은 **파밍 효율과 진행도**를 동시에 결정한다.
* 오프라인 보상 악용을 막기 위해, **오프라인 보상 기준 지역**을 별도로 관리한다.

### 1-3. 시스템 정의

본 문서에서 지역은 아래 요소가 묶인 단위로 정의한다.

* 전투 배경
* 등장 몬스터 그룹
* 보스
* 권장 전투력
* 기본 골드 획득량
* 1회 클리어 보상

### 1-4. 범위

포함:

* 지역 해금
* 지역 진입
* 현재 지역 변경
* 최고 지역 도전
* 클리어/미클리어 상태 관리
* 오프라인 보상 기준 지역 관리

제외:

* 길드전, 펫, 부적, 마정석, 연구소
* 직접 PvP 전투
* 분기형 맵 구조
* 시간 제한 필드 이벤트

## 2. 규칙 설명

### 2-1. 기본 규칙

1. 게임 시작 시 **1지역**이 열려 있다.
2. 지역은 **순차적으로 해금**된다.
3. **N지역 클리어 시 N+1지역 해금**.
4. 플레이어는 아래 지역만 선택할 수 있다.

   * 이미 클리어한 지역
   * 현재 도전 가능한 다음 지역 1개

### 2-2. 지역 상태

각 지역은 아래 4개 상태 중 하나를 가진다.

* **LOCKED**: 아직 입장 불가
* **CHALLENGE**: 입장 가능하지만 미클리어
* **CLEAR**: 클리어 완료, 재입장 가능
* **CURRENT**: 현재 선택된 지역

### 2-3. 지역 진입 규칙

* 플레이어가 지역을 선택하면 서버 검증 후 현재 지역이 변경된다.
* 클리어한 지역은 언제든 자유롭게 이동 가능하다.
* 미클리어 지역은 **최고 클리어 지역 + 1**만 입장 가능하다.
* 잠금 지역 선택 시 입장되지 않으며 안내 문구를 표시한다.

### 2-4. 클리어 규칙

* 해당 지역의 **지역 보스를 1회 처치**하면 클리어 처리한다.
* 클리어 즉시:

  * 최고 클리어 지역 갱신
  * 다음 지역 해금
  * 1회 클리어 보상 지급
* 1회 클리어 보상은 자동 지급한다.

### 2-5. 파밍 규칙

* 현재 선택 지역에서 일반 몬스터 처치 및 반복 전투가 진행된다.
* 플레이어는 막히는 경우 이전 클리어 지역으로 내려가 다시 파밍할 수 있다.
* 지역이 높을수록 기본 골드 획득량과 1회 클리어 보상이 증가한다.

### 2-6. 오프라인 보상 규칙

* 오프라인 보상은 **최대 12시간** 누적된다.
* 오프라인 보상 기준 지역은 `last_valid_farm_region_id`를 사용한다.
* `last_valid_farm_region_id`는 **클리어 완료된 지역 중 마지막으로 확정된 파밍 지역**이다.
* 미클리어 지역에 들어갔다가 종료하더라도, 오프라인 보상은 미클리어 지역 기준으로 계산하지 않는다.

### 2-7. 자동 진행 규칙

* 플레이어는 **자동 진행 ON/OFF**를 설정할 수 있다.
* 자동 진행 ON 상태에서 현재 지역을 클리어하면:

  * 다음 지역으로 자동 진입 시도
  * 단, 자동 진입은 서버 검증 후 적용
* 자동 진행 OFF 상태에서는 클리어 후 현재 지역 유지

### 2-8. 랭킹 반영 규칙

* 지역 선택 시스템에서 랭킹에 직접 반영하는 값은 **최고 클리어 지역**이다.
* PvP 직접 전투는 없으며, 랭킹 시스템은 별도 문서에서 관리한다.

## 3. 데이터 구조

| 필드명                      | 타입     | 설명            |
| ------------------------ | ------ | ------------- |
| region_id                | int    | 지역 ID         |
| chapter_id               | int    | 챕터 ID, 연출용 그룹 |
| order_no                 | int    | 정렬 순서         |
| region_name              | string | 지역명           |
| unlock_region_id         | int    | 선행 클리어 필요 지역  |
| recommended_power        | bigint | 권장 전투력        |
| monster_group_id         | int    | 일반 몬스터 그룹     |
| boss_monster_id          | int    | 지역 보스 ID      |
| bg_asset_id              | string | 배경 에셋 ID      |
| gold_per_sec             | int    | 초당 기본 골드 획득량  |
| first_clear_gold         | int    | 첫 클리어 골드      |
| first_clear_diamond      | int    | 첫 클리어 다이아     |
| first_clear_equip_ticket | int    | 첫 클리어 장비 소환권  |
| first_clear_skill_ticket | int    | 첫 클리어 스킬 소환권  |
| is_active                | bool   | 운영 활성 여부      |

### 3-2. 유저 진행 데이터: `player_region_progress`

| 필드명                       | 타입       | 설명            |
| ------------------------- | -------- | ------------- |
| player_id                 | bigint   | 플레이어 ID       |
| current_region_id         | int      | 현재 선택 지역      |
| highest_cleared_region_id | int      | 최고 클리어 지역     |
| last_valid_farm_region_id | int      | 오프라인 보상 기준 지역 |
| auto_progress_enabled     | bool     | 자동 진행 여부      |
| current_region_entered_at | datetime | 현재 지역 진입 시각   |
| last_region_clear_at      | datetime | 마지막 지역 클리어 시각 |
| updated_at                | datetime | 최종 갱신 시각      |

### 3-3. 로그 데이터: `region_selection_log`

| 필드명                       | 타입       | 설명                             |
| ------------------------- | -------- | ------------------------------ |
| log_id                    | bigint   | 로그 ID                          |
| player_id                 | bigint   | 플레이어 ID                        |
| from_region_id            | int      | 이전 지역                          |
| to_region_id              | int      | 변경 지역                          |
| reason                    | enum     | manual / auto / reconnect / gm |
| player_power              | bigint   | 선택 시점 전투력                      |
| highest_cleared_region_id | int      | 당시 최고 클리어 지역                   |
| created_at                | datetime | 선택 시각                          |

### 3-4. 설계 메모

* 현재 구조는 **선형 진행** 기준이므로 지역별 개별 클리어 테이블 없이도 운영 가능하다.
* 추후 분기형 월드맵이 들어가면 `player_region_clear_history` 테이블을 추가하는 방향으로 확장한다.

## 4. UI 구성

### 4-1. 진입 위치

* 메인 전투 화면의 **지역 버튼**
* 전투 상단의 현재 지역명 탭
* 클리어 직후 다음 지역 팝업

### 4-2. 화면 구성

#### 상단 영역

* 현재 지역명
* 최고 클리어 지역
* 권장 전투력
* 현재 지역 예상 골드 수급량(초당 또는 분당)

#### 중앙 영역

* **세로 스크롤 지역 리스트**
* 각 지역 카드에 아래 정보 표시

  * 지역명
  * 상태 배지 (잠김 / 도전 가능 / 클리어 / 현재)
  * 권장 전투력
  * 1회 클리어 보상 미리보기
  * 현재 선택 여부

#### 하단 영역

* `이전 지역`
* `입장`
* `최고 지역으로 이동`
* `자동 진행 ON/OFF`

### 4-3. 카드 상태 표현

* **LOCKED**: 회색 처리 + 잠금 아이콘
* **CHALLENGE**: 강조 테두리 + “도전 가능”
* **CLEAR**: 일반 밝기 + 체크 표시
* **CURRENT**: 강조 배경 + “현재 전투중”

### 4-4. UX 원칙

* 월드맵 드래그 방식보다 **리스트형 UI**를 우선 사용한다.
* 현재 위치와 다음 목표가 한 화면에서 바로 보이도록 한다.
* 잠금 지역 클릭 시 무반응이 아니라 **해금 조건 안내**를 띄운다.
* 자동 진행 상태는 메인 화면에서도 확인 가능해야 한다.

## 5. 예외 처리

| 상황                   | 처리 방식                                      |
| -------------------- | ------------------------------------------ |
| 잠금 지역 선택             | 지역 변경 불가, “이전 지역 클리어 필요” 메시지 표시            |
| 동일 지역 재선택            | 서버 요청 없이 무시 또는 최소 처리                       |
| 미클리어 지역 상태로 종료       | 오프라인 보상은 `last_valid_farm_region_id` 기준 계산 |
| 클리어 직후 통신 끊김         | 서버 기준으로 1회 클리어 보상 1회만 지급, 재로그인 시 동기화       |
| 자동 진행과 수동 선택이 동시에 발생 | 마지막 서버 승인 요청 기준으로 상태 확정                    |
| 패치로 지역 비활성화          | 접속 시 가장 가까운 유효 클리어 지역으로 강제 이동              |
| 다중 접속(다른 기기 로그인)     | 최신 세션 기준으로 현재 지역 상태 동기화                    |
| 잘못된 지역 ID 요청         | 서버에서 거절, 비정상 요청 로그 적재                      |
| 보상 테이블 누락            | 해당 지역 진입 차단, 운영 알림 발생                      |

### 5-1. 핵심 방어 포인트

* **보상 중복 지급 방지**
* **클라이언트의 임의 지역 점프 방지**
* **오프라인 보상 악용 방지**
* **패치 후 지역 꼬임 복구**

## 6. 서버/클라이언트 처리 구분

### 6-1. 클라이언트 처리

* 지역 선택 화면 렌더링
* 지역 카드 상태 표시
* 버튼 입력 처리
* 선택 중 로딩/UI 피드백
* 지역 변경 후 배경/몬스터 연출 갱신
* 자동 진행 토글 UI 반영

### 6-2. 서버 처리

* 입장 가능 지역 검증
* 현재 지역 변경 저장
* 최고 클리어 지역 판정
* 1회 클리어 보상 지급
* 다음 지역 해금 처리
* `last_valid_farm_region_id` 갱신
* 오프라인 보상 계산 기준 확정
* 비정상 요청 차단 및 로그 적재

### 6-3. 권한 기준

지역 선택 관련 최종 판정은 **서버 권한**으로 처리한다.
클라이언트는 보여주기와 입력 전달만 담당하며, 아래 값은 서버가 확정한다.

* 실제 진입 성공 여부
* 현재 지역
* 최고 클리어 지역
* 1회 클리어 여부
* 오프라인 보상 기준 지역

### 6-4. 추천 API 예시

* `GET /region/list`
* `POST /region/select`
* `POST /region/auto-progress`
* `GET /region/current`

## 7. 로그 및 운영 포인트

### 7-1. 필수 로그

1. **지역 선택 로그**

   * 어느 지역에서 어느 지역으로 이동했는지
   * 수동/자동 여부

2. **지역 클리어 로그**

   * 클리어 시각
   * 클리어 당시 전투력
   * 클리어 소요 시간

3. **지역 정체 로그**

   * 특정 지역에서 장시간 머무는 비율
   * 반복 진입 후 이탈 비율

4. **지역 하향 이동 로그**

   * 상위 지역 도전 후 이전 지역으로 내려온 횟수

5. **오프라인 보상 기준 지역 로그**

   * 실제 누적 시점의 기준 지역
   * 회수 시점의 보상량

### 7-2. 운영 지표

* 지역별 평균 클리어 시간
* 지역별 이탈률
* 지역별 재도전률
* 자동 진행 사용 비율
* 특정 지역 구간의 전투력 부족 비율
* 최고 지역 도달 분포

### 7-3. 운영 활용

* 특정 지역에서 이탈률이 급증하면 난이도 과다 구간으로 판단
* 특정 지역에서 클리어 시간이 지나치게 짧으면 성장 체감 부족 가능성
* 하향 이동이 많은 지역은 권장 전투력 또는 보상 곡선 재조정 대상
* 패치 후 지역별 지표 변화를 비교해 밸런스 안정성 확인

### 7-4. GM/운영 기능

* 플레이어 현재 지역 강제 이동
* 최고 클리어 지역 보정
* 특정 지역 비활성화
* 지역별 보상 배율 임시 조정
* 비정상 클리어/선택 로그 조회

## 한 줄 정리

이 지역 선택 시스템은 **“단순한 선형 구조 + 자유로운 복귀 + 서버 권한 기반 진행 관리”**를 핵심으로 하며, 방치형 게임의 진행 스트레스를 줄이면서도 성장 체감을 유지하는 방향으로 설계한다.

다음 단계로는 이 문서를 기준으로 **지역 마스터 테이블 샘플 20개**와 **화면 와이어프레임 텍스트 버전**까지 이어서 구체화하는 것이 좋습니다.
