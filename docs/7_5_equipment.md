아래는 **단순형 방치 RPG 기준 `장비` 시스템 기획서 초안**이다.
전제는 네가 정한 방향에 맞춰 **4부위 / 랜덤옵션 없음 / 별도 장비 재화 없음 / 합성·초월·세트효과 없음**으로 잡았다.

---

# 장비 시스템 기획서

## 1. 시스템 개요

### 1-1. 시스템 목적

장비 시스템은 유저가 전투력을 가장 직관적으로 체감하는 성장 축이다.
주요 목적은 다음과 같다.

* 몬스터 처치, 소환, 던전 보상을 통해 장비 성장을 반복하게 만든다.
* 4개 부위만으로도 성장 재미를 제공하되, 관리 피로도는 낮춘다.
* 복잡한 옵션 비교 대신 **“더 좋은 장비를 얻고, 장착하고, 강화한다”**는 단순한 루프를 유지한다.
* 장비 강화는 장기 성장축, 장비 획득은 단기 보상축 역할을 맡는다.

### 1-2. 시스템 범위

장비 시스템은 아래 요소만 포함한다.

* 장비 부위: **무기 / 방어구 / 반지 / 목걸이**
* 장비 획득
* 장비 장착 / 비교 / 자동 장착
* 장비 강화
* 장비 중복 처리
* 장비 UI 및 운영 포인트

### 1-3. 핵심 설계 방향

* 장비는 **고정 능력치형**으로 설계한다.
* 랜덤 옵션, 잠금해제 옵션, 잠재능력, 세트효과는 제외한다.
* 장비 인벤토리는 복잡하게 늘리지 않고, **도감형 보유 구조**로 단순화한다.
* 강화는 **장비 개별 강화가 아니라 슬롯 강화**로 처리한다.

  * 예: 무기 강화 +20은 무기를 바꿔도 유지됨
* 중복 장비는 자동으로 처리하여 관리 부담을 줄인다.

## 2. 규칙 설명

## 2-1. 기본 규칙

* 장비는 4부위만 장착 가능하다.
* 각 부위에는 1개씩만 장착할 수 있다.
* 장비는 **장비 ID 기준의 고정 성능**을 가진다.
* 동일 장비는 여러 개 쌓지 않고, 최초 획득 시 해금된다.
* 이미 해금한 동일 장비를 다시 획득하면 자동 보상으로 전환된다.

## 2-2. 장비 능력치 규칙

각 부위는 아래 성격을 가진다.

| 부위  | 핵심 역할 | 대표 능력치 예시      |
| --- | ----- | -------------- |
| 무기  | 공격 성장 | 공격력            |
| 방어구 | 생존 성장 | HP, 방어력        |
| 반지  | 치명 성장 | 치명타 확률, 치명타 피해 |
| 목걸이 | 스킬 성장 | 스킬 피해량, 보스 피해량 |

### 설계 원칙

* 각 장비는 **고정 능력치 묶음**을 가진다.
* 같은 장비는 누구에게나 동일 성능이다.
* 옵션 랜덤 생성은 없다.
* 비교는 “현재 장착 장비 대비 상승/하락” 중심으로 보여준다.

## 2-3. 장비 획득 규칙

장비 획득처는 아래로 제한한다.

* **장비 소환**
* **던전 보상**
* 이벤트 / 퀘스트 / 우편
* 상점 구매 패키지

### 추가 원칙

* **오프라인 보상에서는 장비 완제품 대신 골드/장비 소환권 위주**로 지급한다.

  * 이유: 오프라인 상태에서 장비 드랍 로그가 과도하게 쌓이는 문제를 줄이기 위함
* 지역 진행이 높아질수록 장비 소환권/장비 관련 보상 효율이 상승한다.

## 2-4. 장착 규칙

* 유저는 부위별로 해금한 장비 중 1개를 선택해 장착할 수 있다.
* **자동 장착** 기능을 제공한다.
* 자동 장착 기준은 기본적으로 **종합 전투력 점수**가 가장 높은 장비다.
* 유저가 특정 부위를 수동으로 고정한 경우, 자동 장착이 해당 부위를 덮어쓰지 않는다.

### 전투력 점수 예시

`장비 점수 = 공격력 가중치 + HP 가중치 + 방어력 가중치 + 치명 계수 + 스킬 계수`

운영에서는 이 점수 가중치를 조정해 추천 장착 정확도를 관리한다.

## 2-5. 강화 규칙

강화는 개별 장비가 아니라 **슬롯 단위 강화**다.

예:

* 무기 강화 Lv.10
* 방어구 강화 Lv.7
* 반지 강화 Lv.5
* 목걸이 강화 Lv.3

### 강화 재화

* **골드만 사용**
* 별도 강화석 없음

### 강화 효과

* 무기 강화: 공격력 증가
* 방어구 강화: HP/방어력 증가
* 반지 강화: 치명 관련 수치 증가
* 목걸이 강화: 스킬 관련 수치 증가

### 강화 특징

* 장비를 교체해도 강화 수치는 유지된다.
* 강화 레벨은 부위별로 독립적이다.
* 강화 비용은 레벨이 오를수록 증가한다.
* 특정 구간마다 강화 연출/성장 체감이 강화되도록 연출 포인트를 둔다.

## 2-6. 중복 장비 처리 규칙

* 동일 장비를 다시 획득하면 자동으로 **골드**로 전환한다.
* 이미 더 높은 등급/단계 장비를 보유 중이어도, 낮은 장비를 처음 획득한 경우에는 도감 해금 처리 가능하다.
* 중복 자동 전환 보상량은 장비 희귀도/단계에 따라 다르게 설정한다.

## 2-7. 제외 규칙

본 프로젝트에서는 아래 요소를 장비 시스템에 넣지 않는다.

* 랜덤 옵션
* 장비 합성
* 장비 분해 전용 재화
* 장비 초월
* 세트 효과
* 장비 승급 트리
* 장비 잠재능력
* 장비 프리셋 다중 저장

## 3-3. 계산용 뷰/메모리 구조

### 장비 최종 능력치 계산식

`최종 장비 능력치 = 장착 장비 기본 능력치 + 슬롯 강화 능력치`

### 캐릭터 총 능력치 반영

`캐릭터 총 능력치 = 성장 능력치 + 장비 능력치 + 기타 시스템 버프`

장비는 이 중 하나의 독립 모듈로 계산되며, 서버에서 최종 합산한다.

구성은 아래처럼 단순화한다.

### 중앙 우측

* 선택 장비 상세 정보

  * 하락 능력치: 빨강

### 하단

* 장착 버튼
* 자동 장착 버튼
* 강화 버튼
* 장비 소환으로 이동 버튼

* 전체 / 무기 / 방어구 / 반지 / 목걸이

표시 정보:

* 현재 강화 레벨
* 다음 강화 레벨 효과
* 필요 골드
* 강화 후 예상 전투력 상승량

버튼:

* 1회 강화
* 연속 강화 가능 시 일괄 강화

## 4-4. 자동 장착 UX

* 최초 획득 장비가 현재보다 강하면 팝업 또는 미니 토스트 제공
* 자동 장착 ON 상태면 즉시 교체
* 수동 잠금된 슬롯은 제외
* 자동 교체 발생 시 변경 내역을 로그성 토스트로 보여줌

## 5. 예외 처리

| 상황                          | 처리 방식                        |
| --------------------------- | ---------------------------- |
| 이미 해금한 동일 장비 획득             | 장비 해금 없이 sell_gold 만큼 골드 지급  |
| 장착 불가능한 슬롯에 장착 요청           | 서버 reject, 클라이언트 에러 메시지 표시   |
| 해금하지 않은 장비 장착 요청            | 서버 reject                    |
| 강화 재화 부족                    | 강화 불가, 부족 재화 팝업 표시           |
| 강화 최대치 도달                   | 강화 버튼 비활성화                   |
| 자동 장착 도중 잠금 슬롯 존재           | 잠금 슬롯은 유지, 나머지 슬롯만 비교 갱신     |
| 현재 장착 장비가 삭제/비활성 데이터로 바뀜    | 서버가 기본 대체 장비 또는 직전 유효 장비로 보정 |
| 데이터 불일치로 클라 전투력과 서버 전투력이 다름 | 서버 값 우선, 클라 재동기화             |
| 네트워크 재요청으로 강화 중복 호출 발생      | 서버에서 idempotency 또는 중복 요청 차단 |
| 보상 지급 도중 장비와 골드 지급이 섞임      | 서버에서 트랜잭션 단위로 일괄 처리          |

* 슬롯 강화 효율 곡선
* 자동 장착 추천 점수 가중치
* 지역/던전별 장비 관련 보상량

## 7-4. 운영 리스크

* 중복 장비 보상이 너무 낮으면 소환 체감이 나빠짐
* 강화 효율이 너무 높으면 장비 교체 재미가 약해짐
* 강화 효율이 너무 낮으면 골드 소모처로만 느껴짐
* 반지/목걸이 계열의 점수 계산이 부정확하면 자동 장착 신뢰가 무너짐

# 한 줄 정리

이 장비 시스템의 핵심은 **“장비는 얻는 재미, 강화는 쌓는 재미”**로 분리하는 것이다.
그래서 **장비는 도감형 해금**, **강화는 슬롯 유지형**, **중복은 자동 골드 전환**으로 처리하는 구조가 가장 단순하고 운영도 쉽다.

다음으로는 **#16 소환**을 같은 포맷으로 붙이면 장비 시스템과 자연스럽게 이어진다.
