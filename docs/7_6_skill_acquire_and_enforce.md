전제는 다음과 같습니다.

* 이 문서는 **스킬의 획득, 보유, 중복 처리, 강화**만 정의합니다.
* **전투 장착, 슬롯 구성, 프리셋, 자동 사용 규칙**은 **#21 스킬 세팅 시스템**에서 별도로 정의합니다.
* 재화 제한 정책에 맞춰 **스킬 성장에는 골드 + 동일 스킬 중복분**만 사용합니다.
* 스킬 획득의 주 경로는 **스킬 소환권 기반 획득**이며, 상세 뽑기 연출/확률은 **#16 소환**에서 정의합니다.

# #15 스킬(획득/강화) 시스템 기획서

## 1. 시스템 개요

### 1-1. 목적

스킬 시스템은 플레이어가 전투용 스킬을 수집하고, 중복 획득을 통해 성장시키며, 장기 플레이 동기를 느끼게 하는 계정 단위 성장 축이다.
장비와 별개로 “희귀 스킬 획득”과 “기존 스킬 강화”의 재미를 제공하며, 지역 진행/던전/이벤트/시즌 패스와 자연스럽게 연결된다.

### 1-2. 시스템 범위

이 시스템이 담당하는 범위는 다음과 같다.

* 스킬 획득
* 신규 스킬 해금
* 중복 스킬 누적
* 스킬 강화
* 강화 수치 반영
* 스킬 도감/보유 상태 표시

이 시스템이 담당하지 않는 범위는 다음과 같다.

* 전투에 어떤 스킬을 장착하는지
* 장착 순서/프리셋/사용 여부
* 전투 중 스킬 발동 로직

### 1-3. 핵심 플레이 흐름

**스킬 소환/보상 획득 → 신규 스킬 해금 또는 중복 누적 → 골드와 중복분을 사용해 강화 → 강화된 스킬을 #21에서 장착하여 전투 효율 상승**

### 1-4. 오픈 조건

* 튜토리얼 종료 후 오픈
* 초기부터 플레이어가 성장 요소를 명확히 체감하도록 빠르게 노출
* 단, 세부 뽑기 기능은 #16 소환 오픈 조건을 따름

### 1-5. 설계 방향

* 복잡한 재료 체계를 두지 않는다.
* 강화 실패, 파괴, 하락은 없다.
* 같은 스킬을 여러 번 얻는 것이 곧 성장 재료가 된다.
* 스킬의 “사용 방식”은 바꾸지 않고, **수치만 성장**시킨다.
* 스킬 장착/운용과 획득/강화를 명확히 분리한다.

## 2. 규칙 설명

## 2-1. 스킬 상태 정의

스킬은 플레이어 기준으로 아래 3가지 상태를 가진다.

1. **미보유**

   * 아직 획득하지 못한 상태
   * 리스트에는 잠금 상태로 노출 가능

2. **보유**

   * 1회 이상 획득한 상태
   * 레벨 1부터 시작
   * 중복 획득분을 누적 가능

3. **최대 강화**

   * 최대 레벨 도달 상태
   * 이후 동일 스킬 추가 획득 시 보상 변환 처리

---

## 2-2. 스킬 획득 규칙

### 획득 경로

스킬은 아래 경로로 획득할 수 있다.

* #16 소환에서 획득
* 시즌 패스 보상
* 이벤트 보상
* 퀘스트 보상
* 우편 지급
* 운영 보상/점검 보상

### 획득 결과 처리

#### 1) 미보유 스킬 획득 시

* 해당 스킬이 해금된다.
* 기본 레벨은 **Lv.1**
* “NEW” 표시가 활성화된다.
* #21 장착 시스템에서 선택 가능 상태가 된다.

#### 2) 보유 중인 스킬 획득 시

* 해당 스킬의 **중복 보유 수량**이 증가한다.
* 중복분은 강화 재료로 사용된다.

#### 3) 최대 강화 스킬 획득 시

* 중복분으로 쌓지 않고 즉시 보상 변환 처리한다.
* 변환 재화는 **골드**로 고정한다.
* 변환량은 등급별 운영 테이블로 관리한다.

이유는 간단하다.
플레이어가 “꽝”처럼 느끼지 않게 하면서도, 새로운 별도 재화를 만들지 않기 위해서다.

---

## 2-3. 스킬 강화 규칙

### 강화 조건

스킬 강화는 아래 조건을 만족해야 가능하다.

* 해당 스킬을 보유 중일 것
* 최대 레벨이 아닐 것
* 요구 골드를 보유할 것
* 요구 중복 수량을 보유할 것

### 강화 재료

* **골드**
* **동일 스킬 중복 수량**

추가 재료는 사용하지 않는다.

### 강화 방식

* 강화 1회당 **레벨 1 증가**
* 강화 성공 확률은 **100%**
* 강화 실패/하락/파괴 없음
* 현재 레벨과 다음 레벨의 증가 수치를 미리 표시

### 강화 효과

강화 시 다음 중 해당 스킬의 정의된 수치만 증가한다.

* 피해량 계수
* 회복량
* 버프 수치
* 디버프 수치
* 지속시간
* 발동 확률

반대로 다음 요소는 원칙적으로 강화로 변경하지 않는다.

* 스킬의 기본 메커니즘
* 타겟 방식
* 범위 타입
* 사용 가능 여부
* 스킬 분류
* 장착 슬롯 규칙

즉, **강화는 성능 상승이지 기능 변화가 아니다.**

### 강화 수치 구조

실제 수치는 밸런스 테이블에서 관리하며, 시스템 구조는 아래를 따른다.

* 최대 레벨: 예시 기준 **Lv.10**
* 강화 비용: **등급 + 목표 레벨 기준 테이블 관리**
* 중복 요구량: **목표 레벨 기준 증가형 곡선**
* 골드 요구량: **등급별 배수 적용**

예시 구조:

| 목표 레벨 | 필요 중복 수량 |
| ----- | -------: |
| Lv.2  |        1 |
| Lv.3  |        1 |
| Lv.4  |        2 |
| Lv.5  |        2 |
| Lv.6  |        3 |
| Lv.7  |        3 |
| Lv.8  |        4 |
| Lv.9  |        5 |
| Lv.10 |        6 |

위 수치는 예시이며, 실제 값은 운영 테이블에서 조정한다.

---

## 2-4. 강화 후 반영 규칙

* 강화 성공 즉시 서버 기준 스킬 레벨이 갱신된다.
* 이미 #21에서 장착 중인 스킬이라면 **다음 전투 계산부터 즉시 반영**된다.
* 장착 상태 자체는 바뀌지 않는다.
* 강화만으로 자동 장착되지는 않는다.

---

## 2-5. 정렬/표시 규칙

스킬 리스트는 기본적으로 아래 우선순위로 정렬한다.

1. 강화 가능 스킬
2. NEW 스킬
3. 보유 스킬
4. 미보유 스킬
5. 등급 높은 순
6. ID 또는 표시 순서

필터는 아래 정도만 제공한다.

* 전체
* 보유
* 강화 가능
* 미보유
* 등급별

복잡한 세부 필터는 넣지 않는다.

## 2-6. 타 시스템 연동 규칙

* **#16 소환**: 스킬 획득 결과를 전달
* **#21 스킬 세팅**: 보유/레벨 정보 참조
* **시즌 패스 / 이벤트 / 퀘스트 / 우편**: 직접 스킬 또는 스킬 소환권 지급
* **랭킹/전투 계산**: 최종 스킬 레벨 기반 성능 반영

* 강화 연출

  * 던전

## 5. 예외 처리

| 상황              | 처리 방식                                |
| --------------- | ------------------------------------ |
| 미보유 스킬 강화 요청    | 클라이언트 버튼 비활성화, 서버는 `NOT_OWNED` 반환    |
| 중복 수량 부족        | 강화 불가, 부족 수량 표시                      |
| 골드 부족           | 강화 불가, 골드 획득처 안내                     |
| 최대 레벨 도달        | MAX 처리, 추가 강화 요청 차단                  |
| 최대 레벨 스킬 추가 획득  | 즉시 골드 변환                             |
| 클라이언트 정보가 오래됨   | 서버가 최신 데이터 반환, 클라이언트 강제 갱신           |
| 네트워크 끊김 중 강화 요청 | 요청 결과 확정 전 재요청 금지, 재접속 후 동기화         |
| 동일 요청 중복 전송     | 서버 idempotency 키 또는 txn_id로 중복 처리 방지 |
| 라이브 제외된 스킬      | 신규 획득 차단, 기존 보유 스킬은 정상 표시 가능         |
| 데이터 불일치         | 서버값 우선, 클라이언트 로컬 값 폐기 후 재수신          |

### 중요 처리 원칙

강화/획득은 모두 **서버 권한형**으로 처리한다.
클라이언트는 성공을 가정하고 먼저 확정 표시하지 않는다.

## 6. 서버/클라이언트 처리 구분

## 6-1. 서버 담당

서버는 아래를 책임진다.

* 스킬 획득 결과 확정

## 7-4. 운영 상 주의점

1. **강화 비용이 너무 높으면**

   * 획득의 기쁨은 있는데 성장 체감이 약해진다.

2. **중복 요구량이 너무 낮으면**

   * 금방 최대 강화에 도달해 장기 동기가 약해진다.

3. **오버플로우 골드 전환량이 너무 높으면**

   * 후반 경제 인플레이션이 발생한다.

4. **신규 획득 난도가 너무 높으면**

   * 컬렉션 재미보다 중복 피로가 먼저 온다.

즉, 이 시스템은 **“새로운 스킬을 얻는 기쁨”과 “중복도 의미 있는 성장으로 느껴지는 구조”** 사이 균형이 핵심이다.

# 정리

이 문서 기준으로 #15 스킬 시스템은 아래처럼 정의된다.

* **획득**: 소환/보상으로 스킬 직접 획득
* **보유 처리**: 미보유면 해금, 보유 중이면 중복 누적
* **강화**: 골드 + 동일 스킬 중복 소비
* **성장 방식**: 실패 없이 레벨 상승, 수치만 강화
* **분리 원칙**: 장착/프리셋은 #21에서 처리
* **운영 포인트**: 신규 획득률, 중복 체감, 골드 소모량, 장착률까지 함께 본다

다음 순서는 이 문서와 연결되는 **#21 스킬 세팅/프리셋 시스템 기획서**를 같은 포맷으로 분리 작성하는 것입니다.
