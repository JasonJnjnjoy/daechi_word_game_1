복잡도를 줄이기 위해 **무료/프리미엄 2트랙, 일일/시즌 미션 2종, 28일 시즌, 30티어** 기준으로 설계했어.

# 시즌 패스 시스템 기획서

## 1. 시스템 개요

### 1-1. 시스템 목적

시즌 패스는 유저가 매일 접속하고 핵심 루프를 반복하도록 유도하는 **중기 성장 보상 시스템**이다.
유저는 게임 플레이를 통해 시즌 포인트를 획득하고, 누적 포인트에 따라 티어 보상을 수령한다.

### 1-2. 시스템 역할

시즌 패스는 다음 역할을 수행한다.

* 지역 진행, 장비 강화, 스킬 성장, 던전 플레이를 자연스럽게 유도
* 일일 접속 및 반복 플레이 유지
* 무료 유저에게도 명확한 보상 목표 제공
* 프리미엄 패스를 통해 과금 포인트 제공
* 이벤트/길드와 함께 시즌 단위 보상 회수 동기 형성

### 1-3. 핵심 컨셉

* 시즌 단위 누적 보상형 콘텐츠
* 직접 전투 콘텐츠가 아니라 **플레이 누적량을 보상으로 환산**
* 과도한 분기 없이 단순 구조 유지
* 핵심 루프와 연결된 행동만 미션으로 사용

### 1-4. 기본 구조

* 시즌 기간: **28일**
* 티어 수: **30티어**
* 티어 해금 조건: **티어당 시즌 포인트 100**
* 총 필요 포인트: **3,000**
* 보상 트랙: **무료 / 프리미엄**
* 미션 종류: **일일 미션 / 시즌 미션**
* 해금 시점: **튜토리얼 종료 후 오픈**

### 1-5. 재화 사용 원칙

시즌 패스 보상은 공통 설계 기준에 맞춰 아래 재화만 사용한다.

* 골드
* 다이아
* 장비 소환권
* 스킬 소환권
* 던전 티켓
* 길드 코인

※ 시즌 포인트는 시즌 패스 내부 진행값이며, 일반 재화 인벤토리에 쌓이는 소비형 재화가 아니다.

## 2. 규칙 설명

## 2-1. 진행 규칙

유저는 미션 완료를 통해 시즌 포인트를 얻는다.
누적 시즌 포인트가 일정 수치에 도달하면 해당 티어가 해금되고, 보상을 수령할 수 있다.

### 기본 계산식

* `누적 시즌 포인트 = 완료한 미션 보상 포인트 총합`
* `도달 티어 = floor(누적 시즌 포인트 / 100)`

예시:

* 350포인트 보유 시 → 3티어 도달
* 990포인트 보유 시 → 9티어 도달
* 3000포인트 이상 시 → 30티어 완료

---

## 2-2. 보상 트랙 규칙

### 무료 트랙

* 모든 유저가 기본적으로 이용 가능
* 티어 도달 시 무료 보상 수령 가능

### 프리미엄 트랙

* 별도 상품 구매 시 활성화
* 이미 도달한 티어의 프리미엄 보상도 **소급 수령 가능**
* 시즌 종료 후에는 구매 불가

---

## 2-3. 미션 구성 규칙

구조 단순화를 위해 미션은 2종만 사용한다.

### 일일 미션

* 매일 00:00(KST) 초기화
* 당일 플레이 동기 제공
* 수동 갱신/재추첨 없음

### 시즌 미션

* 시즌 전체 기간 동안 1회 또는 누적 달성
* 장기 목표 제공
* 시즌 종료 시 초기화

---

## 2-4. 미션 설계 원칙

시즌 패스 미션은 반드시 핵심 루프 행동과 연결한다.

사용 가능한 대표 조건:

* 로그인
* 몬스터 처치
* 오프라인 보상 수령
* 장비 소환 횟수
* 스킬 소환 횟수
* 던전 클리어 횟수
* 특정 지역 도달

사용하지 않는 조건:

* PvP 직접 승패
* 길드전
* 펫/부적/마정석/연구소 관련 행동

---

## 2-5. 오프라인 보상 연동 규칙

오프라인 플레이로 누적된 처치 수나 수익은
**유저가 접속 후 오프라인 보상을 수령하는 시점에 시즌 미션 진행도에 반영**한다.

즉,

* 앱 미접속 상태에서는 실시간 반영되지 않음
* 오프라인 보상 정산 시 서버가 한 번에 합산 처리

이 구조는 방치형 게임의 핵심 루프와 가장 자연스럽게 연결된다.

---

## 2-6. 예시 미션 세트

### 일일 미션 예시

| 미션     | 조건             |        보상 |
| ------ | -------------- | --------: |
| 일일 접속  | 1회 접속          | 시즌 포인트 20 |
| 몬스터 처치 | 몬스터 1,000마리 처치 | 시즌 포인트 30 |
| 소환 진행  | 장비 또는 스킬 소환 5회 | 시즌 포인트 20 |
| 던전 도전  | 던전 1회 클리어      | 시즌 포인트 30 |

일일 총 공급량: **100 포인트**

### 시즌 미션 예시

| 미션        | 조건              |         보상 |
| --------- | --------------- | ---------: |
| 지역 진입 I   | 지역 10 도달        | 시즌 포인트 100 |
| 지역 진입 II  | 지역 20 도달        | 시즌 포인트 150 |
| 지역 진입 III | 지역 30 도달        | 시즌 포인트 200 |
| 장비 수집     | 장비 소환 50회       | 시즌 포인트 100 |
| 스킬 수집     | 스킬 소환 50회       | 시즌 포인트 100 |
| 던전 정복     | 던전 20회 클리어      | 시즌 포인트 150 |
| 방치 성장     | 오프라인 보상 20회 수령  | 시즌 포인트 100 |
| 누적 사냥     | 몬스터 50,000마리 처치 | 시즌 포인트 200 |

시즌 총 공급량 예시: **1,100 포인트**

### 공급량 기준

* 시즌 최대 티어 필요 포인트: 3,000
* 총 공급 포인트 예시: 3,900
* 공급률: 약 **130%**

즉, 유저가 일부 일일 미션을 놓쳐도 패스를 완주할 수 있게 설계한다.

---

## 2-7. 보상 지급 규칙

* 티어 도달 후 해당 보상은 수동 수령
* `받기` 또는 `전체 받기` 지원
* 프리미엄 미구매 상태에서는 무료 보상만 수령 가능
* 프리미엄 구매 후, 이미 도달한 티어의 프리미엄 보상은 일괄 수령 가능

---

## 2-8. 시즌 종료 규칙

시즌 종료 시 다음 규칙을 따른다.

* 시즌 포인트 적립 중단
* 미션 진행 중단
* 도달한 티어 중 **미수령 보상은 우편으로 자동 지급**
* 미도달 티어 보상은 소멸
* 다음 시즌은 별도 시즌 데이터로 시작
* 이전 시즌 데이터는 조회용 로그만 유지

## 4. UI 구성

## 4-1. 진입 경로

* 메인 화면 상단 또는 좌측 메뉴에 **시즌 패스 버튼**
* 보상 수령 가능 시 **레드닷 표시**
* 이벤트 메뉴와 별개로 상시 노출

---

## 4-2. 메인 화면 구성

시즌 패스 메인 화면은 구현 난이도를 낮추기 위해 **2탭 구조**로 구성한다.

### 탭 1: 보상

구성 요소:

* 시즌명
* 남은 기간
* 현재 티어 / 최대 티어
* 현재 시즌 포인트
* 무료/프리미엄 보상 열
* 티어별 보상 리스트
* `받기`, `전체 받기`
* 프리미엄 구매 버튼

### 탭 2: 미션

구성 요소:

* 일일 미션 탭
* 시즌 미션 탭
* 진행도 바
* 완료 상태 표시
* 수령 가능 상태 표시

---

## 4-3. UI 상세 요소

### 상단 정보 영역

* 시즌명
* 시즌 종료까지 남은 시간
* 현재 포인트
* 현재 티어

### 티어 보상 리스트

* 세로 스크롤 방식 권장
* 각 티어마다 무료/프리미엄 보상 2행 표시
* 현재 도달 티어 강조
* 이미 수령한 보상은 비활성 처리

### 미션 카드

* 미션명
* 진행도 텍스트 (예: 320/1000)
* 완료 상태
* 수령 버튼
* 미완료 상태에서는 진행도 바 노출

### 프리미엄 팝업

* 구매 혜택 요약
* 프리미엄 트랙 보상 미리보기
* 구매 후 즉시 수령 가능한 보상 수 표시

## 4-4. 레드닷 규칙

레드닷은 아래 조건 중 하나라도 만족하면 노출한다.

* 완료했지만 미수령인 미션 존재
* 도달했지만 미수령인 티어 보상 존재
* 프리미엄 구매 후 소급 수령 가능한 보상 존재

* 추가 패스 경험치 아이템 사용
* 다중 패스 동시 운영
* 업적형 별도 패스 탭
* 전용 패스 상점

## 5. 예외 처리

| 상황                         | 처리 방식                        |
| -------------------------- | ---------------------------- |
| 시즌 종료 직후 미수령 보상 존재         | 도달한 티어의 미수령 보상만 우편 자동 지급     |
| 시즌 종료 후 프리미엄 구매 시도         | 구매 불가 처리                     |
| 동일 보상을 연속 클릭               | 서버에서 중복 수령 방지, 1회만 인정        |
| 수령 중 네트워크 끊김               | 서버 반영 여부 기준으로 재조회 후 UI 갱신    |
| 일일 초기화 시점에 화면 열려 있음        | 서버 시간 기준으로 목록 재로딩            |
| 미션 진행도가 한 번에 목표 초과         | 초과분은 버리고 목표값까지만 인정           |
| 오프라인 보상 수령과 동시에 다수 미션 완료   | 서버에서 일괄 반영 후 결과 패킷 반환        |
| 프리미엄 구매는 성공했으나 보상 일괄 수령 실패 | 구매 상태만 우선 저장, 재접속 시 다시 수령 가능 |
| 길드 미가입 상태에서 길드 코인 보상 수령    | 정상 지급, 추후 길드 해금 후 사용 가능      |
| 잘못된 시즌 설정(티어 누락, 보상 누락)    | 시즌 오픈 차단 또는 운영 경고 로그 발생      |

### 추가 정책

* 시즌 패스는 **서버 시간 기준**으로만 동작한다.
* 클라이언트 시간은 표시용으로만 사용한다.
* 우편 지급 실패 시 재시도 큐에 적재한다.

* 보상 수령 후 서버 응답 기준으로 리스트 갱신
* 일일 초기화 시점 이후 첫 진입 시 전체 재동기화

* 핵심 보상 배치 티어

---

## 7-4. 운영 밸런싱 원칙

* 총 공급 포인트는 최대 티어 필요량의 **120~140%** 권장
* 초반 5티어는 빠르게 열리게 설계
* 중반부터는 플레이 누적량이 체감되게 조정
* 최종 티어 보상은 다이아 또는 소환권 중심으로 배치
* 프리미엄 전용 보상은 강한 가속감을 주되, 필수 기능 해금형 보상은 넣지 않음

# 한 줄 정리

이 시즌 패스는 **핵심 루프를 반복하면 자연스럽게 성장 보상을 회수하는 단순한 누적 보상 시스템**이며,
구현 난이도는 낮추고 운영 조정 여지는 충분히 남겨둔 구조다.

원하면 다음에는 이걸 이어서 **“시즌 패스 실제 보상표 30티어 예시”**까지 바로 작성해줄게.
